# 原子级开发任务清单

> 本文件用于替代粗粒度任务执行。任何模型接手开发时，应优先读取：
>
> 1. `spec.md`
> 2. `ui-interaction.md`
> 3. `ui-implementation-guardrails.md`
> 4. 本文件
>
> 执行规则：
>
> - 每个任务都是尽量小的原子任务。
> - 每完成一个任务，将 `[ ]` 改为 `[x]`。
> - 每个任务必须满足“完成定义”。
> - 涉及 UI 的任务必须同时通过 `ui-implementation-guardrails.md` 中的检查。
> - 不允许跳过兼容性和降级要求。
>
> ## 强制嵌入规则：UI / 测试 / 验收必须进入每个功能原子任务
>
> 从本文件生效后，任何新增或继续拆解的功能任务，不允许只写“开发某功能”。每个功能原子任务必须至少包含：
>
> - 功能交付物：代码、接口、页面、配置或文档。
> - UI 要求：如该任务涉及前端页面/组件/交互，必须列出对应 `ui-interaction.md`、`ui-implementation-guardrails.md` 与 N 章节检查项；如不涉及 UI，必须写明“不涉及 UI”。
> - 测试要求：必须列出单元测试、接口测试、组件测试、E2E、构建检查或“不适用原因”。
> - 验收标准：必须列出用户可观察的验收结果、边界场景、失败/空态/权限态。
> - 标记要求：功能完成后，同一任务内的开发、UI、测试、验收全部满足，才允许把 `[ ]` 改为 `[x]`。
>
> 独立的 N 章节和 `testing-acceptance.md` 仍然保留，但它们是“总规范/总清单”；执行时必须把相关要求映射进具体任务。
