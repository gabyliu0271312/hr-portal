# UI 实施守则与合规检查清单

> 目的：确保任何模型开发 UI 时，都会按照已评估的交互方案实施，而不是自由发挥或照搬早期 ASCII 原型。

## 1. 强制阅读顺序

所有涉及 UI 的开发任务开始前，必须阅读：

1. `spec.md`
2. `ui-interaction.md`
3. 本文件 `ui-implementation-guardrails.md`
4. `atomic-tasks.md` 中对应任务

如果未阅读，不应开始写 UI 代码。

---

## 2. UI 总原则

> 分层流转优先于 ETL：涉及数据开发、清洗、聚合、物化、发布时，UI 必须先表达 ODS → DWD → DWS → ADS 的层级、当前资产位置和下一步产物，再展示具体配置表单。

### 2.1 应该做

- 数据仓库只做资产、建模、指标、治理。
- 数据连接/UCP 只做系统、资源、凭证、连接器、Pipeline、事件、监控。
- 数据仓库中涉及 UCP 的地方只展示状态、引用和跳转。
- 所有复杂表单必须分步或分区。
- 所有破坏性操作必须显示影响分析。
- 一期可视化建模必须是半可视化，不做自由画布。
- 一期指标管理必须是指标口径目录，不做复杂计算引擎。
- 一期血缘必须以引用关系/影响分析为主，不强制图谱。

### 2.2 禁止做

- 禁止在数据仓库中做 UCP 凭证编辑。
- 禁止在数据仓库中做 Pipeline 编排器。
- 禁止在数据仓库中做事件触发器配置。
- 禁止同时新增“数据连接”和“连接器平台”两个一级菜单。
- 禁止把“接口管理”继续作为长期系统设置核心入口。
- 禁止在可视化建模 V1 暗示支持自由拖拽连线。
- 禁止在指标管理一期实现未经评审的 SQL 执行器。
- 禁止绕过影响分析直接删除已被引用字段。

## 3. 菜单合规检查

开发菜单/路由时必须满足：

- [ ] 顶层菜单包含 `数据仓库`。
- [ ] 数据仓库子入口包含：数据资产、数据建模、指标管理、数据治理。
- [ ] 数据仓库下不出现完整“数据连接”配置页。
- [ ] 数据连接/UCP 复杂配置入口不复制到数据仓库。
- [ ] 如果需要展示来源配置，使用“跳转数据连接配置”。
- [ ] 无权限用户看不到对应菜单。
- [ ] 超级管理员可看到全部新增菜单。

不合规示例：

```text
数据仓库 > 数据连接 > 凭证配置
```

正确示例：

```text
数据仓库 > 数据资产 > 来源配置：跳转到数据连接
```

## 4. 数据仓库首页合规检查

首页必须包含：

- [ ] 数据表数量。
- [ ] 数据模型数量。
- [ ] 指标数量。
- [ ] 质量告警数量。
- [ ] ODS/DWD/DWS/ADS 分层概览。
- [ ] 涉及数据开发时展示当前层级、目标层级和产物资产类型。
- [ ] 最新动态或空状态。

首页不得：

- [ ] 不得把连接器数量作为主指标卡。
- [ ] 不得放凭证/Pipeline/事件配置入口作为主操作。

---

## 5. 数据资产页合规检查

资产列表必须有以下筛选：

- [ ] keyword。
- [ ] 分层。
- [ ] 主题域。
- [ ] 来源系统。
- [ ] 负责人。
- [ ] 状态。
- [ ] 质量状态。

资产列表必须有以下列：

- [ ] 表名。
- [ ] 显示名。
- [ ] 分层。
- [ ] 主题域。
- [ ] 来源系统。
- [ ] 负责人。
- [ ] 字段数。
- [ ] 最近同步时间。
- [ ] 质量状态。
- [ ] 状态。
- [ ] 操作。

资产详情必须分区：

- [ ] 基础信息。
- [ ] 数仓属性。
- [ ] 来源信息。
- [ ] 运行信息。
- [ ] 治理信息。

UCP 来源展示要求：

- [ ] UCP 未启用：显示“数据连接平台未启用”。
- [ ] UCP 已启用且有关联：显示跳转按钮。
- [ ] 不显示任何凭证明文。

---

## 6. 字段定义页合规检查

字段表格必须有：

- [ ] 字段编码。
- [ ] 字段名称。
- [ ] 数据类型。
- [ ] 主键。
- [ ] 敏感。
- [ ] 维度/度量。
- [ ] 来源。
- [ ] 可见。
- [ ] 描述。
- [ ] 操作。

字段详情必须使用抽屉或等价侧边详情，不建议用超大弹窗。

字段详情必须分区：

- [ ] 基础信息。
- [ ] 数仓属性。
- [ ] 权限属性。
- [ ] 变更影响。

破坏性操作要求：

- [ ] 删除字段前调用影响分析。
- [ ] 修改字段类型前调用影响分析。
- [ ] blocking=true 时阻止操作或要求强确认。

---

## 7. 快速关联合规检查

快速关联必须是三步：

- [ ] Step 1：选择主表和关联表。
- [ ] Step 2：配置关联条件。
- [ ] Step 3：选择输出字段并预览。

必须支持：

- [ ] 保存草稿。
- [ ] 发布为模型。
- [ ] 预览前 20 条。
- [ ] 输出字段改名。

不得：

- [ ] 不得把快速关联做成复杂自由画布。

---

## 8. 可视化建模 V1 合规检查

V1 页面布局必须是：

- [ ] 顶部：模型基础信息。
- [ ] 左侧：数据表选择区。
- [ ] 中间：关系图展示区。
- [ ] 右侧：节点/连线配置区。
- [ ] 底部：输出字段配置与预览。

V1 必须支持：

- [ ] 添加表。
- [ ] 移除表。
- [ ] 多表链式关联。
- [ ] 自动展示节点和连线。
- [ ] 点击节点编辑表配置。
- [ ] 点击连线编辑关联配置。
- [ ] 输出字段配置。
- [ ] 预览。
- [ ] 保存草稿。
- [ ] 发布。

V1 不得出现：

- [ ] 字段拖拽到字段创建连线。
- [ ] 节点自由拖拽并保存布局。
- [ ] 缩放/平移作为核心能力。
- [ ] 版本回滚。

如果 UI 中保留这些按钮，必须标记为“二期能力，暂不可用”，且不能影响验收。

---

## 9. 数据预览合规检查

预览区域必须包含：

- [ ] 前 20 条数据表格。
- [ ] 主表行数。
- [ ] 关联后行数。
- [ ] 未匹配行数。
- [ ] 重复匹配行数。
- [ ] 空值数量。
- [ ] 字段类型异常数量。

如果某个统计后端暂不支持：

- [ ] 前端显示 `暂未统计`，不能报错。

---

## 10. 指标管理合规检查

一期指标管理定位：指标口径目录。

列表必须包含：

- [ ] 指标编码。
- [ ] 指标名称。
- [ ] 指标类型。
- [ ] 业务定义。
- [ ] 主题域。
- [ ] 依赖数据集。
- [ ] 负责人。
- [ ] 状态。
- [ ] 版本。
- [ ] 操作。

表单必须分区：

- [ ] 基础信息。
- [ ] 计算口径。
- [ ] 依赖数据。
- [ ] 发布信息。

不得：

- [ ] 不得在一期新增复杂 SQL 执行器。
- [ ] 不得让用户误以为指标会自动计算，除非后端已实现。

建议提示文案：

```text
当前指标管理用于统一业务口径和依赖关系，一期不执行复杂指标计算。
```

---

## 11. 数据治理合规检查

一期治理页必须聚焦：

- [ ] 数据分层。
- [ ] 影响分析。
- [ ] 基础质量检查入口。

治理工作台可展示：

- [ ] 未设置分层的数据表。
- [ ] 未设置负责人的模型。
- [ ] 最近字段变更。
- [ ] 高风险删除操作。
- [ ] 质量检查失败项。

不得强制实现：

- [ ] 完整字段级血缘图谱。
- [ ] 质量规则调度趋势图。
- [ ] 复杂告警中心。

---

## 12. 数据连接/UCP 协同合规检查

数据仓库页面中涉及 UCP 时：

- [ ] 只展示 UCP 资源引用。
- [ ] 只展示最近测试/同步状态。
- [ ] 只提供跳转按钮。
- [ ] UCP 未启用时友好降级。

不得在数据仓库中实现：

- [ ] 凭证编辑。
- [ ] Adapter 注册。
- [ ] Pipeline 编排。
- [ ] 事件总线配置。
- [ ] 熔断/重试深度配置。

---

## 13. 每次 UI 交付必须填写的自检说明

任何模型完成 UI 相关任务后，最终回复或提交说明中必须包含：

```text
UI 合规自检：
- 是否阅读 ui-interaction.md：是/否
- 是否阅读 ui-implementation-guardrails.md：是/否
- 涉及页面：...
- 是否引入了禁止项：否/说明
- 可视化建模是否仍为 V1 半可视化：是/不涉及
- 是否在数据仓库中重复实现 UCP 配置：否/说明
- 破坏性操作是否接入影响分析：是/不涉及
```

如果无法满足，应明确说明原因，不得假装完成。
