# 自动化数仓开发三期规划 — ODS→DWD 自动化 + 指标驱动 + L4 全自动级联

> 本文件定义自动化数仓开发能力的三期建设路线，按优先级分三期交付。
>
> **第一期（本期优先）**：ODS→DWD 自动化级联 — ODS 数据源同步完成后自动触发 DWD 标准化，消除人工"同步完再去配方构建器点执行"的断点。
>
> **第二期**：指标驱动的自动化数仓开发能力（原 X05 完整内容）— 指标定义 → DWS/ADS 草稿生成与受控发布。
>
> **第三期（远期受控试点）**：L4 全自动级联发布 — 指标保存后自动建表、调度、更新 ADS/BI，严格受控，默认禁用。
>
> 执行规则与模板沿用 `atomic-tasks.md` 的 A 章约定。本章节新增的 UI 示意图编号从 U28 起。

# Z. 自动化数仓开发三期原子任务

## Z00 三期实现契约与禁止项（Z 章评审必查）

### 功能需求说明与目的

**功能定位：** Z 章定义自动化数仓开发能力的三期路线图，从不建"无人值守全自动级联"起步，逐期提升自动化程度，每期都以人工确认为最终安全边界。

**要解决的问题：**
- 第一期：ODS 表通过 DataSource/UCP 定时同步后，用户仍需手工进入"数据加工配方构建器"点击执行才能更新 DWD，链路存在人工断点。
- 第二期：业务用户配置指标后，仍需数据开发人员手工创建 DWS/ADS，链路长、易出错。
- 第三期：在积累足够信任和审计能力后，对低风险指标开放全自动级联试点。

**目标用户：** 数据管理员、数据建模人员、指标负责人、报表开发者。

**本期要实现（Z 章整体）：** 三期递进，每期独立可交付、独立可回滚。

**本期不实现：**
- 不在第一期就开放全自动级联。
- 不跳过人工确认直接改生产 ADS。
- 不建设独立调度中心（复用 `app/scheduler/`）。

**与平台化的关系：** 自动化能力是平台从"手工工具"走向"受控自动化服务"的关键路径，但必须以可审计、可回滚、可关闭为前提。

### 能力分级

| 期数 | 级别 | 能力 | 实现原则 |
|------|------|------|----------|
| 第一期 | L1-ODS | ODS 同步完成 → 自动触发 DWD 标准化 | Feature flag 控制，默认关闭 |
| 第一期 | L2-ODS | ODS→DWD 自动化规则配置 + 预览 + 审计 | 用户显式配置关联关系，每次执行留审计 |
| 第二期 | L1 | 指标定义生成 DWS/ADS 草稿 | 必须人工确认 |
| 第二期 | L2 | 发布时生成或刷新 View/物化结果 | 必须有预览、影响分析和回滚 |
| 第二期 | L3 | 指标变更生成下游 ADS/BI 影响方案 | 默认只生成更新建议 |
| 第三期 | L4 | 指标保存后全自动建表、调度、更新 ADS/BI | 仅作为远期受控试点，feature flag 默认关闭，需审批 |

### 红线（三期通用）

1. ❌ 不绕过权限、血缘、质量门禁做自动化。
2. ❌ 不在 ODS 层做业务口径加工。
3. ❌ 不提供任意 SQL/脚本编辑器。
4. ❌ 不跳过 DWS 直接 ODS→ADS。
5. ❌ 不新建第二套调度系统。
6. ❌ L4 全自动级联默认关闭，仅在审批通过后对试点空间开放。

# 第一期：ODS→DWD 自动化级联

## Z01 ODS → DWD 自动化级联

### 功能需求说明与目的

**功能定位：** Z01 在第一期最优先级解决 ODS 数据源同步完成后自动触发 DWD 标准化的核心痛点。当前链路为：DataSource 定时同步 ODS → 用户收到通知 → 打开"数据加工配方构建器" → 手动点击"执行" → 发布 DWD 视图。Z01 将"用户手动点击执行"替换为"用户在配置页一次性声明 ODS 表与标准化规则的关联关系，之后每次 ODS 同步完成自动触发 DWD 标准化执行"。

**要解决的问题：** ODS 定时同步已由 `datasource_sync` handler 全自动完成，但 DWD 标准化仍需人工介入，导致数仓分层流转出现"半自动"断点。Z01 消除这个断点。

**目标用户：** 数据建模人员、数据管理员。

**核心使用场景：**
- 数据建模人员在 ODS 表上配置好标准化规则后，打开"自动化配置"面板，将此 ODS 表关联到已配置的规则集。
- 选择触发策略："ODS 同步成功后自动执行"或"手动执行（默认）"。
- 每次 DataSource 定时同步或手动拉取完成后，系统自动检查该 ODS 表是否配置了自动化规则，如有则执行标准化并更新 DWD 视图/表。
- 用户可在审计页查看每次自动触发的执行记录、成功/失败状态和影响行数。

**本期要实现：**
- `datasource_sync_completed` 标准事件注册。
- `trigger_dwd_standardization` 自动化动作注册。
- ODS 同步完成后的事件发布（在 `sync_service.py` 中）。
- ODS→DWD 自动化规则配置 UI（在数据加工配方构建器内或数据资产详情页）。
- Feature flag 控制，默认关闭。
- 自动执行审计记录。

**本期不实现：**
- 不自动创建标准化规则（规则仍需人工配置）。
- 不自动跳过质量门禁。
- 不在 ODS 层做业务口径加工。
- 不将自动化级联到 DWS/ADS（那是第二期、第三期的范围）。

**与现有能力的关系：**
- 复用 `app/scheduler/` 通用调度组件。
- 复用 `app/automation/` 事件/规则引擎。
- 复用 `app/warehouse/service/standardization.py` 的 `execute_full()`。
- 复用 `app/warehouse/standardization_engine.py`。
- 不新建调度表、不新建事件总线。

  - 完成定义：专项验收通过后，才允许向试点用户开放 ODS→DWD 自动化入口。

# 第二期：指标驱动的自动化数仓开发能力

> 以下内容完整保留原 `## X05 指标驱动的自动化数仓开发能力`，包括全部原子任务（X0501-X0513）和 U27 线框图。
> 本期在 Z01（第一期 ODS→DWD 自动化）已交付的基础上，建设"指标定义驱动 DWS/ADS 自动生成草稿与受控发布"的高级能力。

## X05 指标驱动的自动化数仓开发能力

### 功能需求说明与目的

**功能定位：** X05 在三期已具备 ODS→DWD→DWS→ADS 受控加工、视图/物化、血缘、权限继承、影响分析和发布能力，以及**第一期 Z01 已实现 ODS→DWD 自动化级联**的基础上，建设"指标定义驱动 DWS/ADS 自动生成草稿与受控发布"的高级能力。该能力用于弥补当前没有独立数仓开发平台的缺口，但仍必须保持数据仓库模块的受控边界：系统可以自动生成候选方案和草稿，生产发布必须经过预览、校验、影响分析和人工确认。

**要解决的问题：** 业务用户配置指标后，仍需要数据开发人员手工理解口径、创建 DWS 汇总视图、组装 ADS 消费资产并告知帆软/BI 使用，链路长、易出错、难追踪。X05 目标是把三期沉淀的受控加工组件编排起来，降低重复配置成本，同时避免"保存指标即自动改生产表/自动影响报表"的高风险模式。

**目标用户：** 数据管理员、指标负责人、数据建模人员、报表开发者、权限管理员、审计人员。

**核心使用场景：** 指标负责人在指标管理中维护指标定义、维度、过滤条件和依赖字段；系统生成 DWS 聚合视图草稿与 ADS 消费视图草稿；用户预览数据、查看 SQL 摘要、血缘、权限继承、小样本风险和下游影响；确认后发布为数据资产；帆软/BI 定期读取已发布 ADS View 或 DWS View 时看到新数据。

**本期要实现：** 指标定义解析、DWS/ADS 草稿生成、受控 SQL/View 生成、预览与影响分析、权限继承校验、小样本风险校验、人工确认发布、版本/回滚、运行审计、BI/帆软消费提示和降级状态。

**本期不实现：** 不实现任意 SQL/脚本开发器；不做复杂 DAG 调度中心；不在保存指标时无确认地自动建表、自动发布或自动改 ADS；不自动修改帆软报表设计；不替代 UCP 跨系统 Pipeline；不绕过权限、血缘、质量门禁和审批直接开放敏感数据。

**与平台化的关系：** X05 是"没有独立数仓开发平台"情况下的数据仓库高级自动化能力，但其产物仍是平台可治理的数据资产、视图、发布记录和消费契约，而不是 HR Portal 内部临时脚本。未来若独立数仓平台或 UCP 数据开发能力成熟，X05 应可迁移为外部平台能力，HR Portal 保留指标治理和消费入口。

**与第一期 Z01 的关系：** Z01 已在 ODS 同步完成后自动完成 ODS→DWD 标准化，使 DWD 层数据保持"准实时"新鲜度。X05 在此基础上从指标定义出发，自动生成 DWS/ADS 草稿，用户确认后发布。两期串联后完整链路为：

```
ODS 定时同步（datasource_sync）
  ↓ Z01 自动触发
DWD 标准化 + 视图刷新
  ↓
指标定义解析（X0502）
  ↓ X05 自动生成草稿
DWS 草稿 → 人工确认 → DWS 发布
  ↓ X05 自动生成草稿
ADS 草稿 → 人工确认 → ADS 发布
  ↓
帆软/BI 定期读取
```

**能力分级：**

| 级别 | 能力 | 本期处理原则 |
|---|---|---|
| L1 | 指标定义生成 DWS/ADS 草稿 | 本期优先实现，必须人工确认 |
| L2 | 发布时生成或刷新 View/物化结果 | 可实现，必须有预览、影响分析和回滚 |
| L3 | 指标变更生成下游 ADS/BI 影响方案 | 可实现，默认只生成更新建议 |
| L4 | 指标保存后全自动建表、调度、更新 ADS/BI | **移至第三期**，仅作为远期受控试点候选 |

# 第三期：L4 全自动级联发布（远期受控试点）

## Z03 L4 全自动级联发布

### 功能需求说明与目的

**功能定位：** Z03 在 Z01（ODS→DWD 自动化）和 X05（指标驱动 DWS/ADS 草稿生成与受控发布）已稳定运行、积累充分信任和审计能力后，对**低风险指标**开放 L4 全自动级联发布试点。这是自动化数仓开发的最终形态：指标保存 → 自动建表 → 自动调度 → 自动更新 ADS/BI。

**要解决的问题：** 前两期仍存在人工确认环节（DWS 发布确认、ADS 发布确认），对高频变更的低风险指标（如月度常规 KPI、固定口径统计指标）来说，人工确认成为吞吐瓶颈。L4 在严格受控范围内消除这些确认环节。

**目标用户：** 数据管理员（审批试点资格）、指标负责人（配置全自动规则）、审计人员（监控全自动执行）。

**核心使用场景：**
- 管理员在系统设置中为特定空间/指标类型开启 L4 试点。
- 指标负责人为低风险指标配置全自动规则：选择触发条件（指标保存后 / ODS 同步完成后 / DWD 更新后）、配置最大自动发布频率、设置自动回滚条件。
- 系统在指标保存后自动执行：解析 → 生成 DWS 草稿 → 通过所有门禁 → 自动发布 DWS → 生成 ADS 草稿 → 通过所有门禁 → 自动发布 ADS → 更新 BI 消费契约。
- 任一门禁失败（质量/小样本/影响分析高风险）则自动阻断并通知指标负责人。
- 全量执行记录可审计、可一键回滚。

**本期要实现：**
- L4 feature flag + 审批流程（默认关闭）。
- 指标级自动化规则配置（替代全局 L4 开关，粒度到单个指标）。
- 全自动级联执行引擎（复用 Z01 的事件驱动 + X05 的草稿/发布服务）。
- 自动门禁检查与阻断逻辑（质量/小样本/影响分析/权限/敏感字段）。
- 全链路审计时间线。
- 一键回滚能力（撤回 DWS + ADS + BI 消费契约到上一个版本）。
- 紧急停止开关（全局 + 按指标）。

**本期不实现：**
- 不将 L4 作为默认行为（必须审批后开启）。
- 不对高风险指标（涉及敏感字段、大范围下游影响、新口径未经评审）开放 L4。
- 不自动修改帆软报表设计。
- 不绕过审批直接开放。

**与平台化的关系：** L4 是"受控自动化数据平台"的最终能力证明，但仅对经过验证的低风险场景开放。它证明了平台既能自动化，也能治理自动化本身。

# 三期总结与依赖关系

```text
第一期（Z01）：ODS→DWD 自动化级联
  ├── Z0101 feature flag
  ├── Z0102 datasource_sync_completed 事件
  ├── Z0103 trigger_dwd_standardization 动作
  ├── Z0104 自动化配置 API
  ├── Z0105 sync_service 事件发布
  ├── Z0106 自动化配置 UI（U28）
  ├── Z0107 审计与可观测性
  └── Z0108 专项验收
        ↓
第二期（X05）：指标驱动的自动化数仓开发
  ├── X0501 feature flag
  ├── X0502-X0512 原子任务
  ├── X0513 专项验收
  └── U27 线框图
        ↓
第三期（Z03）：L4 全自动级联发布
  ├── Z0301 feature flag + 审批
  ├── Z0302 指标级规则配置
  ├── Z0303 L4 执行引擎
  ├── Z0304 紧急停止 + 回滚
  ├── Z0305 全链路审计
  ├── Z0306 专项验收
  └── U29 线框图
```

**能力递进关系：**

| 期数 | 自动化范围 | 触发条件 | 人工介入点 |
|------|-----------|----------|-----------|
| 第一期 | ODS→DWD | ODS 同步完成（事件驱动） | 配置自动化规则（一次性） |
| 第二期 | DWD→DWS→ADS | 用户手动触发"生成草稿" | DWS 发布确认 + ADS 发布确认 |
| 第三期 | ODS→DWD→DWS→ADS 全链路 | 指标保存 / ODS 同步完成 | 开通试点（一次性审批）、门禁阻断时人工介入 |

**不可越级：**
- 第一期未稳定运行前，不可启动第二期（因为 X05 假设 DWD 数据已通过自动化保持新鲜）。
- 第二期未通过专项验收前，不可启动第三期（因为 L4 依赖 X05 的所有服务）。
- 第三期仅对已通过第一期+第二期验收的低风险指标开放。
