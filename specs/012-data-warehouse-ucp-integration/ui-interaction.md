# UI 交互设计需求与评估

> 本文档是 `spec.md` 的 UI 交互细化文件。开发 UI 时必须遵循本文档，而不是直接照搬早期 ASCII 原型。

## 1. UI 总体评估结论

早期 UI 原型覆盖面完整，但存在边界混合问题：

- 数据仓库与数据连接/UCP 混在同一平台视角下。
- “数据连接”和“连接器平台”并列，用户心智重复。
- “接口管理”放在系统设置中，长期不适合 UCP 资源管理。
- 数据血缘、可视化建模、指标计算等能力原型偏最终态，一期实现风险过高。

修正后的 UI 原则：

```text
数据仓库管资产、模型、指标、治理。
数据连接/UCP 管系统、资源、凭证、连接器、Pipeline、事件、监控。
两者通过资源引用、状态展示、跳转、Bridge Adapter 打通。
```

## 2. 顶层导航

### 2.1 推荐导航

```text
首页 | 数据仓库 | 数据连接 | 提效工具 | 业务应用 | 系统设置
```

### 2.2 数据连接命名

- 如果与 UCP 合并后作为一级菜单，菜单名使用 `数据连接`。
- 页面标题可显示 `数据连接平台`。
- 不要同时出现 `数据连接` 和 `连接器平台` 两个一级菜单。

### 2.3 系统设置中的接口管理

长期应迁移为：

```text
数据连接 > 系统资源 / 资源管理
```

迁移期可以保留旧入口，但应重定向或提示“已迁移到数据连接”。

## 3. 数据仓库首页

### 3.1 页面目标

帮助用户快速了解：

- 有多少数据资产。
- 数据分布在哪些层。
- 哪些模型和指标已发布。
- 是否存在质量/同步异常。
- 最近有哪些数据变更。

### 3.2 必须包含的卡片

```text
数据表数量
数据模型数量
指标数量
质量告警数量
```

可选：

```text
最近同步失败数
待治理资产数
未设置负责人资产数
```

### 3.3 分层概览

展示 ODS/DWD/DWS/ADS 四层：

```text
ODS 原始数据层
DWD 明细数据层
DWS 汇总数据层
ADS 应用数据层
```

每层展示：

- 表/模型数量。
- 最近更新时间。
- 异常数量。

### 3.4 最新动态

动态类型：

- 数据同步成功/失败。
- 模型发布。
- 字段变更。
- 指标发布。
- 质量检查失败。
- 下游引用变更。

### 3.5 不应出现

首页不应把“连接器数量”作为主要指标。连接器属于数据连接/UCP。

### 4.2 操作

每行操作：

```text

```

`来源配置` 规则：

- 如果有关联 UCP 资源，跳转到数据连接资源详情。

- 如果无来源，显示“手工维护/未配置”。

### 4.3 详情页/抽屉

表资产详情包含：

```text

运行信息：最近同步时间、最近同步状态、行数
治理信息：质量状态、影响分析入口
```

- 显示引用列表。
- 阻止删除或要求管理员二次确认。
- 对已发布模型/报表引用字段，默认禁止删除。

## 7. 可视化建模 V1

### 7.1 定位

V1 是半可视化建模，不做完整自由画布。

### 7.2 页面布局

```text
顶部：模型基础信息
左侧：数据表选择区
中间：关系图展示区
右侧：关联/节点配置区
底部：输出字段配置与数据预览
```

### 7.3 顶部基础信息

字段：

```text
模型名称
分层：ODS/DWD/DWS/ADS
主题域
负责人
模型描述
业务口径
状态：草稿/已发布/停用
```

### 7.4 左侧数据表选择区

功能：

- 搜索表。
- 按分层/主题域/来源筛选。
- 添加表到模型。
- 显示已选表。

### 7.5 中间关系图

V1 要求：

- 自动根据 DataSetTable/DataSetRelation 渲染节点和连线。
- 节点显示表名、别名、关键字段。
- 连线显示 join_type 和关联字段摘要。
- 点击节点在右侧显示表配置。
- 点击连线在右侧显示关联配置。

V1 不要求：

- 字段拖拽连线。
- 节点自由拖拽持久化。
- 缩放和平移。
- 复杂自动布局算法。

### 7.6 右侧配置区

节点配置：

```text
表别名
表角色：主表/维表/事实表
说明
```

连线配置：

```text
左表别名
右表别名
关联类型
关联字段组
基数
```

### 7.7 底部输出字段

字段：

```text
来源表
来源字段
输出编码
输出名称
数据类型
维度/度量
是否敏感
是否显示
描述

```

### 7.8 数据预览


## 11. 可用性要求

1. 所有复杂表单使用分步或分区，避免单页过长。
2. 所有发布动作必须有草稿状态。
3. 所有破坏性操作必须展示影响分析。
4. 所有预览操作必须有加载态、失败提示、空状态。
5. 所有从数据仓库跳转到 UCP 的入口必须在 UCP 未启用时显示友好提示。
6. 权限不足时隐藏操作按钮，不仅依赖后端报错。
7. 表格列较多时使用列设置或抽屉详情，避免横向滚动过重。

## 追加：评审修订后的 UI 交互要求

### UCP 摘要与跳转

数据仓库页面可以展示 UCP 资源摘要，但不能内嵌 UCP 复杂配置：

- 可展示：UCP 系统名称、资源名称、资源类型、最近测试状态、最近执行状态、跳转入口。
- 不可展示：凭证编辑、Pipeline 画布、事件触发器、死信/重试深度配置。
- 无 UCP 权限时：仍可查看资产详情中的只读摘要；编辑配置时跳转 UCP 并由 UCP 权限拦截。
- UCP 未启用时：显示“数据连接平台未启用，当前资产使用现有 DataSource 同步”，不能影响现有同步入口。

### 首页质量和动态口径

一期首页只展示可落地指标：

- 质量状态：来自 `last_quality_status`。
- 最近同步：来自 `SyncRun`。
- 模型/指标发布：来自 `published_at`。
- 字段变更：来自字段元数据更新时间。

不得展示没有后端来源的“实时质量告警数量”。

### 轻量 ELT 入口占位

一期允许出现灰度/占位入口，但必须明确状态：

- 数据集“物化构建”：一期可显示禁用按钮或“后续支持”。
- 指标“物化计算”：一期可显示规划态字段，不触发计算。
- 质量规则“运行检查”：一期如未实现执行，必须禁用并说明。
- 快照/拉链：一期不展示主入口，最多在文档或高级规划中说明。
