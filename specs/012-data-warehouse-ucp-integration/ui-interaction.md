# UI 交互设计需求与评估

> 本文档是 `spec.md` 的 UI 交互细化文件。开发 UI 时必须遵循本文档，而不是直接照搬早期 ASCII 原型。

## 1. UI 总体评估结论

早期 UI 原型覆盖面完整，但存在边界混合问题：

- 数据仓库与数据连接/UCP 混在同一平台视角下。
- “数据连接”和“连接器平台”并列，用户心智重复。
- “接口管理”放在系统设置中，长期不适合 UCP 资源管理。
- 数据血缘、可视化建模、指标计算等能力原型偏最终态，一期实现风险过高。

修正后的 UI 原则：

```text
数据仓库管资产、模型、指标、治理。
数据连接/UCP 管系统、资源、凭证、连接器、Pipeline、事件、监控。
两者通过资源引用、状态展示、跳转、Bridge Adapter 打通。
```

## 2. 顶层导航

### 2.1 推荐导航

```text
首页 | 数据仓库 | 数据连接 | 提效工具 | 业务应用 | 系统设置
```

### 2.2 数据连接命名

- 如果与 UCP 合并后作为一级菜单，菜单名使用 `数据连接`。
- 页面标题可显示 `数据连接平台`。
- 不要同时出现 `数据连接` 和 `连接器平台` 两个一级菜单。

### 2.3 系统设置中的接口管理

长期应迁移为：

```text
数据连接 > 系统资源 / 资源管理
```

迁移期可以保留旧入口，但应重定向或提示“已迁移到数据连接”。

## 3. 数据仓库首页

### 3.1 页面目标

帮助用户快速了解：

- 有多少数据资产。
- 数据分布在哪些层。
- 哪些模型和指标已发布。
- 是否存在质量/同步异常。
- 最近有哪些数据变更。

### 3.2 必须包含的卡片

```text
数据表数量
数据模型数量
指标数量
质量告警数量
```

可选：

```text
最近同步失败数
待治理资产数
未设置负责人资产数
```

### 3.3 分层概览

展示 ODS/DWD/DWS/ADS 四层：

```text
ODS 原始数据层
DWD 明细数据层
DWS 汇总数据层
ADS 应用数据层
```

每层展示：

- 表/模型数量。
- 最近更新时间。
- 异常数量。

### 3.4 最新动态

动态类型：

- 数据同步成功/失败。
- 模型发布。
- 字段变更。
- 指标发布。
- 质量检查失败。
- 下游引用变更。

### 3.5 不应出现

首页不应把“连接器数量”作为主要指标。连接器属于数据连接/UCP。

## 11. 可用性要求

1. 所有复杂表单使用分步或分区，避免单页过长。
2. 所有发布动作必须有草稿状态。
3. 所有破坏性操作必须展示影响分析。
4. 所有预览操作必须有加载态、失败提示、空状态。
5. 所有从数据仓库跳转到 UCP 的入口必须在 UCP 未启用时显示友好提示。
6. 权限不足时隐藏操作按钮，不仅依赖后端报错。
7. 表格列较多时使用列设置或抽屉详情，避免横向滚动过重。

## 8. 评审修订后的 UI 交互要求

### UCP 摘要与跳转

数据仓库页面可以展示 UCP 资源摘要，但不能内嵌 UCP 复杂配置：

- 可展示：UCP 系统名称、资源名称、资源类型、最近测试状态、最近执行状态、跳转入口。
- 不可展示：凭证编辑、Pipeline 画布、事件触发器、死信/重试深度配置。
- 无 UCP 权限时：仍可查看资产详情中的只读摘要；编辑配置时跳转 UCP 并由 UCP 权限拦截。
- UCP 未启用时：显示“数据连接平台未启用，当前资产使用现有 DataSource 同步”，不能影响现有同步入口。

### 首页质量和动态口径

一期首页只展示可落地指标：

- 质量状态：来自 `last_quality_status`。
- 最近同步：来自 `SyncRun`。
- 模型/指标发布：来自 `published_at`。
- 字段变更：来自字段元数据更新时间。

不得展示没有后端来源的“实时质量告警数量”。

### 分层流转与轻量 ELT 入口占位

一期允许出现 ODS → DWD → DWS → ADS 分层流转视图或灰度/占位入口，但必须明确状态：

- 数据集“物化构建”：一期可显示禁用按钮或“后续支持”。
- 指标“物化计算”：一期可显示规划态字段，不触发计算。
- 质量规则“运行检查”：一期如未实现执行，必须禁用并说明。
- 快照/拉链：一期不展示主入口，最多在文档或高级规划中说明。

## 9. 系统设置归并 UI 原则

字段管理、同步历史、表间关联、数据视图迁移到数据仓库过程中，旧入口必须提供清晰迁移提示：

- 当前入口是否只读。
- 新入口在哪里。
- 是否保留旧数据。
- 是否有下线时间。
- 用户下一步应该点击哪个按钮。

接口配置从系统设置迁出；资产级拉取/推送/API 暴露融入数据仓库“来源与开放”，平台级连接器/凭证/接口定义/Pipeline 跳转 UCP/数据连接。

所有 UI 线框图以内置在 `atomic-tasks.md` 的 U 章节为准。

## 10. 数据接入、资产卡片与表间关联授权 UI 修订

## 11. T02 接口配置拉取/推送融合交互评审

### 1. 评审结论

接口配置不能简单从系统设置迁移到 UCP，也不能原样搬到数据仓库。现状中它同时包含：

- 拉取接口：DataSource / SyncRun。
- 推送接口：PushTarget / JobRun / API 暴露。

数据仓库应从“资产生命周期”视角融合这些能力，形成资产详情中的“来源与开放”入口。

### 2. 交互原则

1. **整体系统内不冗余**：用户不应同时看到多个同名“接口配置”主入口。旧入口应标记为兼容/全局运维入口。
2. **无 UCP 时不降级**：没有数据连接/UCP 时，现有拉取、推送、API 暴露、调度和历史仍可配置和运行。
3. **有 UCP 时不重复建设**：UCP 负责复杂连接器、凭证、资源、Pipeline；数据仓库展示摘要、轻操作、跳转和治理影响。
4. **不是页面搬家**：数据仓库不复制旧接口配置页面，而是在资产详情中围绕“入仓来源 / 出仓目标 / 影响与治理”组织信息。

### 3. 资产详情推荐交互

在资产详情增加 `来源与开放` Tab：

- 入仓来源 Pull Sources：展示 DataSource / UCP Resource。
- 出仓目标 Push Targets：展示 HTTP 推送、外部库、API 暴露、飞书表格。
- 影响与治理：展示字段变更影响、质量门禁、权限/脱敏风险。

主操作：

- 测试连接。
- 立即同步。
- 立即推送。
- 查看同步/推送历史。
- 字段映射。
- 编辑轻量配置。
- 跳转 UCP/数据连接。

### 4. 旧入口兼容交互

系统设置旧接口配置入口在兼容期保留：

- 顶部展示迁移提示。
- 列表增加方向标签：拉取 / 推送 / API 暴露。
- 列表增加 owner 标签：DataSource / PushTarget / UCP。
- 资产相关项优先跳转到数据仓库资产详情的 `来源与开放` Tab。
- 全局运维场景仍可编辑当前 DataSource/PushTarget，直到数据仓库融合入口具备等价能力。

线框图见 `atomic-tasks.md`：`U20 接口配置拉取/推送融合线框图`。

### 5. 系统设置旧入口最终下线原则

接口配置不应长期留在系统设置。交互上必须体现三阶段：

1. 兼容期：旧入口可访问，但顶部提示“接口配置正在迁移到数据仓库 > 来源与开放”；资产相关项优先跳转数据仓库。
2. 主迁移期：旧入口降级为只读/全局搜索/跳转；新增和编辑优先进入数据仓库资产详情，平台级能力跳转 UCP/数据连接。
3. 下线期：隐藏系统设置菜单，旧 URL 重定向到数据仓库来源与开放或显示清晰下线说明。

数据仓库的“来源与开放”必须成为资产级拉取、推送、API 暴露的主入口；系统设置不能继续形成第二个独立编辑入口。

## 12. T05 数据视图与数据资产合并交互方案

### 1. 评审结论

系统设置下的“数据视图”不应长期作为独立模块存在，应迁移并合并到“数据仓库 > 数据资产”的统一资产目录。数据视图是可消费的数据资产，不是系统配置项。

### 2. 信息架构

- 主入口：`数据仓库 > 数据资产`。
- 资产类型：表、数据视图、数据模型、指标、开放 API。
- 旧入口：`系统设置 > 数据视图` 仅作为兼容入口，逐步从可访问提示、只读跳转到最终隐藏/重定向。

### 3. 列表与卡片交互

数据资产目录必须支持：

- 卡片视图：默认给业务用户使用，突出名称、类型、主题域、来源、质量、权限、引用和主操作。
- 表格视图：给管理员批量治理使用，保留批量编辑、批量下线、批量授权或批量导出等治理字段。
- 类型筛选：必须能按“数据视图”过滤。
- 来源筛选：支持旧数据视图迁移、DataSource、UCP 摘要、模型派生、手工登记等来源。

数据视图卡片至少展示：名称、类型标签、分层、主题域、来源、最近构建/同步、质量、字段数/记录摘要、权限状态、引用数、详情/预览/血缘/申请权限等操作。

### 4. 详情交互

数据视图详情复用统一资产详情框架，但必须额外突出：

- 视图定义：来源表/模型、筛选条件、派生字段、版本。
- 数据预览：按权限脱敏；无权限时显示申请入口。
- 权限：授权对象、字段可见性、行级条件、有效期和审计。
- 血缘与影响分析：展示上游表/模型和下游报表/API/推送目标；删除字段、修改类型、归档视图前必须进入影响分析。
- 来源与开放：展示 DataSource/UCP/PushTarget/API 暴露摘要和跳转，不展示 secret。

### 5. 旧入口兼容交互

系统设置旧数据视图入口必须显示迁移 Banner：

- 当前状态：兼容期 / 只读期 / 已下线。
- 新入口：数据仓库 > 数据资产。
- 当前数据视图的新资产详情跳转。
- 迁移说明和下线计划。

旧入口不允许继续新增一套独立数据视图；如兼容期仍保留编辑，必须调用同一个数据仓库 service，不能双写。

### 6. 线框图依据

开发必须以 `atomic-tasks.md` 的 `U21 数据视图与数据资产合并线框图` 为准；资产卡片同时参考 `U18`，旧入口参考 `U14`。如实现与线框图不一致，必须先更新 U 章节再开发。

## 13. 分层数据开发三期与四期高级能力交互方案

### 三期数据开发交互

三期数据开发能力必须先围绕 ODS → DWD → DWS → ADS 分层流转设计，再落到“仓内轻量 ELT”：字段标准化、清洗、DWS 聚合、ADS 组装、物化、调度均只处理已入仓资产。UI 采用受控表单、向导、预览和影响分析，不提供任意 SQL/脚本输入。

- 分层流转总览参考 `atomic-tasks.md` U25。
- ODS → DWD 字段标准化/清洗参考 `atomic-tasks.md` U22。
- DWD → DWS 聚合与 DWS → ADS 组装参考 `atomic-tasks.md` U23。
- 仓内调度继续参考 U12。

### 四期高级能力交互

四期页面首先表达能力归属和状态，而不是直接提供编辑器：

- BI 自助分析：显示“在 BI 中分析”跳转和引用状态，不内建图表/大屏设计器。
- SQL/脚本沙箱：默认禁用；如试点，必须显示审批、资源限额、审计和风险提示。
- 跨系统 Pipeline：显示 UCP 摘要和跳转，不展示可编辑 Pipeline 画布。
- 高级维度模型：仅在复评通过后提供受控建模入口。

四期 UI 参考 `atomic-tasks.md` U24；如实现不一致，必须先更新 U 章节线框图和交互说明。

## 14. AI 接入预留的未来 UI 交互边界

> 本节仅约束二期及以后新增 AI UI，不代表一期已实现 AI 入口。具体原子任务见 `atomic-tasks.md` Y 章，线框图见 U26。

### 14.1 入口原则

- 只有完成 capability 注册、权限裁剪、Policy Guard 和审计后，页面才允许出现 AI 入口。
- 入口文案必须具体表达能力边界，例如“解释血缘”“总结质量问题”“生成规则草稿”，不得使用“AI 自动处理”“智能一键发布”等误导性文案。
- 无权限、上下文为空、Policy Guard 拒绝或 UCP 不可用时，入口必须禁用并展示原因。

### 14.2 只读解释抽屉

只读解释类能力用于资产、字段、血缘、质量、指标、影响分析说明。抽屉必须展示：

- capability 名称。
- 当前用户和数据范围。
- 字段隐藏/脱敏/行级范围裁剪摘要。
- AI 输出和引用来源。
- 审计编号。

只读解释不得替代原始数据列表、血缘图、质量规则、影响分析列表和 blocking 判断。

### 14.3 草稿生成面板

草稿生成类能力用于 ODS→DWD 标准化、DWD→DWS 聚合、DWS→ADS 组装、指标和质量规则草稿。面板必须展示：

- “草稿，未保存/未发布”状态。
- 每条建议的来源字段、目标字段、规则类型、敏感继承、风险提示和采纳/编辑/放弃操作。
- 发布前必须进入预览、影响分析、权限校验和人工确认。

### 14.4 高风险能力复评页

写入、发布、导出、开放 API 类 AI 能力在四期复评前默认禁用。UI 只能展示状态、风险、前置条件和申请试点入口，不提供直接执行按钮。

### 14.5 状态要求

所有 AI UI 必须覆盖：normal、loading、empty、error、forbidden、policy-denied、context-truncated、audit-failed。涉及 UCP 摘要时额外覆盖 ucp-unavailable。涉及草稿时额外覆盖 draft-unsaved、draft-applied、draft-discarded。
