1.2 建设边界：哪些在 HR Portal，哪些在帆软
核心原则：HR Portal 做“供给侧”的确定性加工，帆软做“消费侧”的灵活分析。

加工类型	归属	原因
字段标准化（重命名、类型转换、枚举映射）	✅ HR Portal	源头治理，一次加工全公司复用
数据清洗（去重、空值处理、格式标准化）	✅ HR Portal	源头治理，避免脏数据流入下游
多表关联（JOIN 生成宽表）	✅ HR Portal	口径统一，一次关联全公司复用
基础聚合（按维度汇总，如按月+部门汇总人数）	✅ HR Portal	DWS 层建设，标准指标口径
指标计算（按指标定义自动生成 DWS 视图）	✅ HR Portal	指标口径统一管理
ADS 层组装（DWS + 维度表 → 宽表）	✅ HR Portal	最终数据出口，帆软直接读取
数据质量校验（空值/唯一性/业务规则检查）	✅ HR Portal	源头保障
数据血缘追踪	✅ HR Portal	Portal 数据资产治理
复杂维度建模（星型/雪花模型设计）	⭐ 帆软	帆软可视化建模更强
探索式数据处理（一次性分析、临时数据组合）	⭐ 帆软	灵活、按需、不需要固化
复杂计算（同比/环比、占比、排名、累计）	⭐ 帆软	帆软公式引擎更强大
可视化报表制作（图表、仪表板、大屏）	⭐ 帆软	帆软核心强项
自助分析（业务人员拖拽式查询）	⭐ 帆软	帆软核心强项
一句话判断标准：

“这个加工结果会被 3 个以上下游复用吗？”

是 → HR Portal 做（源头治理，一次加工多次消费）

否 → 帆软做（灵活分析，按需处理）

三、字段标准化（本次重点补充）
3.1 什么是字段标准化？
字段标准化是将 ODS 层原始数据中的字段，按照统一规范进行转换的过程，使数据进入 DWD 层时具备一致的命名、类型、编码和格式。

核心目标：让不同来源的数据使用同一套“语言”，确保后续多表关联、指标计算时不会因为字段名不一致、类型不匹配、枚举值不统一而导致混乱或口径错误。

3.2 字段标准化的具体能力
能力一：字段重命名（Rename）
将原始字段名改为统一的命名规范。

原始字段（北森）	标准化字段	规范说明
realName	employee_name	全小写+下划线
empId	employee_id	统一主键命名
deptName	department_name	避免缩写
hireDate	hire_date	日期字段统一后缀
isActive	is_active	布尔字段统一前缀
能力二：数据类型转换（Type Convert）
将字段数据类型统一为规范类型。

原始字段	原始类型	标准化类型	说明
hireDate	VARCHAR	DATE	日期统一用DATE类型
salary	VARCHAR	DECIMAL(10,2)	金额统一用DECIMAL
isActive	INT(1)	BOOLEAN	布尔统一用BOOLEAN
employee_id	VARCHAR(50)	VARCHAR(32)	统一长度
能力三：枚举值映射（Value Map）
将不同系统用不同编码表示的值，映射成统一的业务含义。

原始字段	原始值	标准化值	说明
gender	0 / 1	男 / 女	数字→文字
gender	M / F	男 / 女	字母→文字
status	1 / 2 / 3	在职 / 离职 / 待入职	数字→文字
is_active	1 / 0	true / false	数字→布尔
is_active	Y / N	true / false	字母→布尔
employee_type	1 / 2 / 3	正式员工 / 实习生 / 劳务派遣	数字→文字
能力四：单位统一（Unit Convert）
同一类数据使用统一的计量单位。

原始字段	原始单位	标准化单位	转换规则
salary	千元	元	× 1000
salary	万元	元	× 10000
tenure	年	月	× 12
age	出生日期	岁数	计算当前日期-出生日期
能力五：字段拆分/合并（Split/Merge）
把一个字段拆成多个，或多个字段合并成一个。

原始字段	标准化字段	说明
full_name = "张三"	first_name = "张", last_name = "三"	拆分（如需要）
full_name = "张三"	employee_name = "张三"	保持原样
address = "深圳市南山区"	city = "深圳", district = "南山"	地址拆分
year + month	period = "2026-07"	合并为期间

四、其他数据加工能力的补充说明
4.1 数据清洗（已规划）
能力	说明	示例
去重	按业务主键去除重复数据	employee_id 相同，保留最新一条
空值处理	对空值进行填充、替换或标记	部门为空 → 填充“未分配”
格式标准化	统一日期、电话等格式	2026/07/05 → 2026-07-05
4.2 多表关联建模（V1已实现，V2待开发）
能力	说明	示例
可视化画布	拖拽表到画布，连线定义关联	员工表 + 部门表 + 岗位表 → 员工宽表
多表链式关联	支持三张及以上表的关联	A → B → C → D
关联类型	左连接、右连接、内连接	员工表 LEFT JOIN 部门表
字段输出配置	选择输出的字段，重命名	只输出需要的字段，改为业务名
4.3 DWS 聚合（按指标定义自动生成）
能力	说明	示例
自动生成 SQL	根据指标定义自动生成聚合 SQL	离职率定义 → 自动生成 Group By 语句

能力	说明	示例

预置筛选	针对特定看板预置过滤条件	只显示“在职员工”数据
4.5 数据质量校验
规则类型	说明	示例
空值检查	指定字段不允许为空	employee_id 不能为空
唯一性校验	指定字段值唯一	employee_id 必须唯一
业务规则校验	自定义业务逻辑	离职日期不能早于入职日期
格式校验	字段格式合规	手机号格式、邮箱格式
4.6 数据血缘追踪
能力	说明	示例
