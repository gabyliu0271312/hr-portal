# 权限管控 —— 管理员维护手册

> 面向:系统管理员 / 业务负责人（在前端页面配置权限的人）
> 配套架构文档:《[permission-model.md](./permission-model.md)》
> 版本:v1（2026-06-18,以代码现状 alembic 0037 为准）

本手册只讲**"在哪个页面、配什么、产生什么效果"**,不讲代码实现。四层权限从上到下配置,顺序建议照本手册章节走。

---

## 〇、四层权限速记

| 层 | 一句话 | 维护页面 |
|---|---|---|
| L1 功能权限 | 能进哪些菜单、点哪些按钮 | 角色配置 |
| L2 数据范围 | 能看哪些行(哪些人/部门/成本中心) | 管理单元(标签) + 用户管理 |
| L3 字段权限 | 能看哪些字段、要不要脱敏 | 字段分类 + 全局字段 + 字段管理 |
| L4 资源授权 | 能用哪些报表/数据集 | 报表设计 / 数据集设计 的 ACL |

> 所有维护页本身都受 L1 保护,菜单代码前缀 `system.*` / `datasource.*`,默认只有超管能改。

---

## 一、L1 功能权限 —— 角色配置

**页面**:角色配置　`/system/roles`　(menuCode `system.roles`)

**配什么**:
1. 新建/编辑角色。
2. **菜单权限矩阵**:为角色逐菜单勾选 5 个动作——
   - **V 查看**:能否进这个菜单(不勾=左侧导航看不到、直接输地址被踢回首页)。
   - **C 新增 / U 修改 / D 删除 / E 导出**:对应按钮能否点(不勾=按钮隐藏)。
3. 菜单上的**数据范围维度** `scope_dimension`(组织 / 成本中心 / 无)——决定该菜单走哪套 L2 口径。
4. 把角色绑定给用户(也可在"用户管理"侧绑)。

**效果**:用户的菜单权限 = 其所有角色**取并集**(任一角色给了就有)。

**常见操作**:
- 新人看不到某菜单 → 检查其角色有没有勾该菜单的 V。
- 按钮点不了 → 检查对应动作列(C/U/D/E)。

## 二、L2 数据范围 —— 管理单元(标签) + 分配给用户

### 2.1 建标签

**页面**:管理单元/数据范围标签　`/system/scopes`　(menuCode `system.scopes`)

**一个标签 = 组织范围段 + 人员范围段**,两段在标签内是 **AND**(都满足):

- **组织范围段**:
  - 维度选 **组织树** 或 **成本中心树**。
  - 勾选可管辖的节点;勾"含下级"则该节点所有子孙自动纳入。
  - 勾"无限制"则组织范围这段放行(只受人员段约束)。
- **人员范围段**(可选,启用后):
  - 按 **用工类型 / 用工主体 / 人员** 加筛选条件(等于 / 不等于,可多值)。
  - 多个条件之间 **AND**。

### 2.2 把标签分给用户

**页面**:用户管理　`/system/users`　(menuCode `system.users`)　或在标签页分配。

- 一个用户可绑**多个标签**,多标签之间是 **OR**(命中任一标签即可见)。
- 用户**没绑任何标签 → 看不到任何受控表数据**(默认拒绝)。

### 2.3 效果与口径

- 用户查数据/报表时,系统自动按"标签并集"过滤行,只返回其范围内的数据。
- 组合语义:`(标签1: 组织 AND 人员) OR (标签2: ...) OR ...`。

**举例**:
- "技术中心负责人":1 个标签,组织树勾"技术中心"+含下级,人员段不启用 → 看技术中心及下属所有人。
- "只看正式工的 HRBP":组织段勾管辖部门,人员段加"用工类型 = 正式" → 范围内的正式工。

> ⚠️ 本轮改造后(见架构文档 G1/G3):**所有业务表无例外受控**,工资表等"只有工号"的表也会经花名册按你的组织范围过滤。配错标签会直接看不到数据,优先自查标签。

## 三、L3 字段权限 —— 分类 / 全局字段 / 字段管理

字段权限的核心是**字段分类**:把敏感字段(如薪酬)归到一个"敏感分类",再决定谁能看、哪些工具能特许使用。

### 3.1 建字段分类

**页面**:字段分类　`/system/field-categories`　(menuCode `system.field_categories`)

**配什么**:
1. 新建分类(如"薪酬"),勾 **是否敏感**。
2. 把物理字段挂进分类(指定 表 + 列)。
3. 配 **授权工具白名单**:把"补偿金计算""证明开具"等工具加入该分类——
   - 含义:**无该分类权限的人,在这些工具里仍可使用该分类字段的原值**(因为这正是授权工具的用途)。
   - 不在白名单的通用场景(数据集/自助报表),无权者该字段直接隐藏。

### 3.2 全局字段(跨表统一治理)

**页面**:全局字段　`/system/global-fields`　(menuCode `system.global_fields`)

**配什么**:新建全局字段(如"工号""部门"),设标准名、数据类型、**权限角色 scope_role**(绑哪棵树:组织/成本中心/用工类型/用工主体/人员)、**字段分类**。物理列认领全局字段后,**自动继承其分类与权限角色**。

### 3.3 字段管理(物理列)

**页面**:字段管理　`/system/field-columns`　(menuCode `system.field_columns`)

**配什么**:对每张表的物理列——认领全局字段、标 `scope_role`(参与 L2 过滤的维度列)、标敏感、配枚举值。

### 3.4 谁能看某分类

- **直接授权**:在用户管理把分类授权给用户。
- **角色授权**:把分类授权给角色,角色下用户继承。
- 用户可见分类 = 直接授权 ∪ 角色授权。

### 3.5 效果(裁决顺序)

```
用户对该字段分类有权限?
├─ 有 → 原值可见(任何场景)
└─ 无 → 当前工具在该分类白名单里?
        ├─ 在  → 原值可见(如补偿金/证明)
        └─ 不在 → 字段隐藏(通用数据集/报表里根本不出现)
```

> 本轮改造后(G2):脱敏/隐藏字段的**真值不再从数据库取出**,更安全。展示效果不变(仍是 `******` 或不出现)。

## 四、L4 资源授权 —— 报表 / 数据集 ACL

### 4.1 报表授权

**页面**:报表设计　`/report/designer/:id`(ACL 编辑器)　;列表 `/report/list`(menuCode `report.list`)

**配什么**:在报表 ACL 里添加可访问的**角色 / 用户**。

**可见规则**:创建者、超管恒可见;**已发布(is_published)的报表所有登录用户可见**;否则按 ACL 命中;无 ACL 且未发布 → 仅创建者。

> ⚠️ "已发布"= 对所有登录用户开放该报表资源(行/字段仍受 L2/L3 约束)。不想全员可见就**别发布、用 ACL 精确授权**。

### 4.2 数据集授权

**页面**:数据集设计　`/datasource/datasets/:id`(ACL 编辑器)　;列表 `/datasource/datasets`(menuCode `datasource.datasets`)

**配什么**:数据集 ACL 添加角色 / 用户。**无 is_published 概念**:创建者/超管/ACL 命中可用,否则仅创建者。

## 五、排障速查

| 现象 | 先查哪 |
|---|---|
| 菜单看不到 | L1:角色有没有勾该菜单 V |
| 按钮点不了 | L1:对应动作列 C/U/D/E |
| 表里一行数据都没有 | L2:用户有没有绑标签、标签范围对不对 |
| 某些字段是 `******` 或不显示 | L3:用户缺该字段分类权限;需授权或确认是否该看 |
| 工具里提示无权用某字段 | L3:把该工具加进对应分类的授权工具白名单 |
| 看不到某张报表/数据集 | L4:加 ACL 或(报表)发布 |

## 六、新增权限相关菜单/页面时

参见《新增菜单页面 SOP》(全局 memory):建 Vue → 注册路由(带 `menuCode`)→ 加 `MENU_TREE` → 重启 → 在角色配置里给角色授权。新页面默认只有超管能进,记得授权。
