# 数据集 ACL 升级为全局业务数据消费授权底座

> 文档类型：原子级开发任务文档  
> 目录：`specs/016-dataset-acl-foundation/`  
> 状态：Draft for Review  
> 编写日期：2026-07-21  
> 适用范围：HR Portal 的报表、通用数据查询、数据对比、全局 AI、导出、飞书及后续开放接口

## 1. 背景与目标

- **背景：**
  - HR Portal 已有功能权限、组织/成本中心/人员范围过滤、字段分类权限，以及数据集 ACL。当前报表链路会校验其绑定的数据集 ACL；但通用数据查询和数据对比仍以已注册物理表名为数据入口，未来全局 AI、飞书和开放接口也可能绕过数据集 ACL。
  - 仅依赖行级范围和字段级权限，不能回答“该角色是否应接触这一类数据”。例如 HRBP 即便只能看到负责组织范围内的记录，也不应自动查询薪资、个税、绩效、劳动关系等职责外数据主题。
  - 数据资产目录的浏览权属于数据治理/元数据管理，不应自动等同于读取资产中业务数据的权利。
- **目标：**
  1. 将 `DataSetAcl` 从报表模块内的校验逻辑升级为全局统一的**业务数据消费授权底座**。
  2. 将数据集定义为“面向业务消费的最小可授权数据产品”：它同时约束业务主题、来源表、输出字段、关联关系和数据范围策略。
  3. 面向业务用户的数据消费必须遵循：`功能权限 → 数据集 ACL → 数据集内容约束 → 行级范围 → 字段级权限`。
  4. 普通业务用户与全局 AI 不得从物理表名开始查询；必须先获得目标数据集上下文。
  5. 报表、通用查询、导出、数据对比、全局 AI、飞书和未来开放 API 使用同一数据集授权判定，且授权撤销在下一次请求立即生效。
- **非目标 / 不做范围：**
  - 不处理部门负责人“组织树”与“直属汇报关系”两种团队口径的选择；本期继续复用现有组织树、成本中心树和范围标签。
  - 不建立普通业务用户的独立“角色—物理表 ACL”管理界面。
  - 不开放任意 SQL、任意物理表、任意字段、任意 JOIN 或跨数据集自由组合。
  - 不重写现有 `scope_filter` 的标签合并语义，不替代字段分类、字段隐藏/脱敏和授权工具白名单。
  - 不要求一次性下线管理员维护用的底表能力；但其必须与普通业务消费入口隔离。
  - 本期不实现飞书机器人和开放 API 的具体业务能力；只预留并规定其接入统一授权服务的契约与验收门禁。

## 2. 用户场景

### 2.1 数据管理员：配置可消费的数据集与授权对象

- **入口：** 数据仓库 → 数据模型/数据集设计页。
- **操作：** 创建或编辑数据集，维护业务名称、说明、来源表、关联、输出字段、范围策略、消费状态，并按角色或用户配置 ACL。
- **系统反馈：** 保存前检查至少有一个来源表、关联和输出字段合法、ACL 角色/用户有效；保存后显示该数据集的授权摘要和可消费状态。
- **成功结果：** 目标角色/用户在授权后可发现并消费该数据集；未授权用户不可在业务查询入口发现它。
- **失败/空态/无权限表现：**
  - 无 `datasource.datasets:C/U`：返回 403，前端不展示保存/授权操作。
  - 无来源表、非法关联、重复名称或输出字段不属于数据集：返回 400 并定位错误项。
  - 数据集未处于可消费状态：普通业务用户不可使用；管理员可在设计页看到状态原因。

### 2.2 HRBP：查询获授权组织范围内的人力数据

- **入口：** 通用数据查询入口，后续可由全局 AI 调用同一查询能力。
- **操作：** HRBP 选择“人员异动数据集”，选择已批准字段与筛选项，查询近三个月的人员异动。
- **系统反馈：** 系统仅展示其可访问数据集、该数据集的可用输出字段和已批准筛选维度；结果自动按 HRBP 的组织/成本中心范围过滤，无权字段隐藏或按既有规则脱敏。
- **成功结果：** 仅返回 HRBP 有数据集权限、范围权限和字段权限的结果。
- **失败/空态/无权限表现：**
  - 数据集授权不存在或已撤销：403，不返回数据集字段、物理表名或样例值。
  - 有数据集权限但无范围标签/范围不匹配：200 空结果；不把“无范围”误报为“全量无数据”。
  - 请求字段不属于该数据集或无字段权限：400/403，字段不进入查询、筛选、聚合、排序或 distinct 值。

### 2.3 HRBP：尝试查询未授权薪酬主题

- **入口：** 通用数据查询、全局 AI、数据对比或导出入口。
- **操作：** HRBP 请求“查看负责组织员工的薪资/奖金/个税”。
- **系统反馈：** HRBP 的可消费数据集列表中不出现薪酬数据集；即使输入业务名称、物理表名或字段名，系统也不将其解析为可执行数据源。
- **成功结果：** 不返回任何薪酬记录、薪酬字段、表名、关联建议、样例值或部分统计。
- **失败/空态/无权限表现：** 返回统一无权提示；前端不得用“空表”或字段列表暴露未授权对象。

### 2.4 报表使用者：共享报表不扩大底层数据访问权

- **入口：** 报表列表、详情、运行、复制、CSV/XLSX 导出。
- **操作：** 使用者打开一张被共享的报表。
- **系统反馈：** 除报表可见性规则外，系统再次校验其绑定数据集 ACL、当前范围与字段权限。
- **成功结果：** 有数据集权限者按最新范围和字段权限获得报表结果；授权被撤销后下一次运行或导出被拒绝。
- **失败/空态/无权限表现：** 无底层数据集权限者不能查看、运行、复制或导出该报表数据。

### 2.5 数据对比用户：仅比较两侧均获授权的数据集

- **入口：** 数据对比技能配置、执行页和全局 AI 数据对比能力。
- **操作：** 用户选择两个可比较数据集，配置名单、字段或金额对比。
- **系统反馈：** 系统只展示用户有权的数据集；每一侧均校验数据集 ACL，字段和关联键均从对应数据集已批准输出字段/关联中选择。
- **成功结果：** 两侧均获授权时，分别执行范围和字段裁决后返回差异。
- **失败/空态/无权限表现：** 任一侧未授权时不执行对比，不返回部分明细、未授权表名、字段或关联建议。

### 2.6 数据资产目录浏览者：查看元数据但不可读取业务数据

- **入口：** 数据仓库 → 数据资产。
- **操作：** 用户浏览资产名称、分层、负责人、来源、质量状态和经批准显示的字段元数据。
- **系统反馈：** 系统按 `warehouse.assets:V` 控制资产目录访问；不将此权限转换为数据集 ACL。
- **成功结果：** 用户可完成资产治理信息浏览，但只能消费获数据集 ACL 授权的数据。
- **失败/空态/无权限表现：** 无数据集 ACL 时，查询、AI、对比、导出等业务消费入口不得返回对应业务记录。

## 3. 功能范围

| 功能项 | 是否本期实现 | 说明 |
| --- | --- | --- |
| 统一数据集 ACL 服务 | 是 | 从 `datasets/router.py::_can_access` 提炼为可复用、唯一权威的服务 |
| 可消费数据集发现接口 | 是 | 面向通用查询、数据对比、AI 等入口返回当前用户获授权且可消费的数据集摘要 |
| 数据集消费状态校验 | 是 | 普通业务消费只允许处于可消费状态且启用的数据集 |
| 报表接入统一 ACL 服务 | 是 | 保持现有行为，去除跨模块重复的 ACL 判定 |
| 通用查询改为数据集入口 | 是 | 新增数据集查询 API 与页面；普通用户不再从 `/data/{table}` 读取业务数据 |
| 通用查询导出随数据集授权 | 是 | 导出必须携带 `dataset_id`，复用同一裁决和投影 |
| 数据对比按数据集双侧授权 | 是 | CompareSpec 与存储任务由 `table` 迁移为 `dataset_id` + 数据集内输出字段引用 |
| 全局 AI 数据集安全上下文 | 是 | 只向 AI 暴露当前用户可消费数据集的业务元数据；不实现泛化任意 SQL |
| 飞书/开放 API 统一接入契约 | 是 | 增加授权服务使用规范、测试桩和接入检查；不实现新渠道业务界面 |
| 数据集授权配置与摘要 UI | 是 | 复用现有数据集编辑页，补完整的 ACL 编辑、可消费状态与影响说明 |
| 资产目录与消费权限分离 | 是 | 保持资产浏览能力，不授予业务数据读取；增加前后端提示与回归测试 |
| 直属汇报关系团队范围 | 否 | 继续复用现有范围标签与组织树 |
| 面向普通业务用户的物理表 ACL | 否 | 不作为主授权模型 |
| 任意物理表直接查询 API 对外开放 | 否 | 历史 `/data/{table}` 仅保留给受控管理员维护路径，普通业务入口迁移后不再调用 |

## 4. 技术设计

### 4.1 数据库 / 数据模型

#### 4.1.1 现有表复用

| 表 | 本期用途 | 现状/改造 |
| --- | --- | --- |
| `datasets` | 数据集主体；业务名称、状态、范围策略、创建人 | 复用 `is_active`、`status`、`scope_strategy`；新增消费状态口径统一函数，不新增重复状态列 |
| `dataset_acl` | 数据集按角色/用户的访问授权 | 复用；补唯一约束和有效性校验，防止同一授权重复写入 |
| `dataset_tables` | 数据集可使用的物理表及别名 | 仅供服务端数据集执行器解析；不可直接暴露为普通业务用户查询入口 |
| `dataset_relations` | 数据集内部批准的关联关系 | 通用查询、数据对比和 AI 只能使用该集合 |
| `dataset_output_fields` | 数据集对消费方发布的输出字段 | 普通查询、对比、AI 的可选字段白名单来源 |
| `reports` / `report_acl` | 报表资源与共享规则 | 保留；报表始终额外校验 `dataset_id` 的数据集 ACL |
| `registered_tables` / `table_columns` | 数据资产和底表元数据 | 继续用于管理员治理与内部建模；不作为普通业务消费授权来源 |
| `data_compare_tasks` / `ai_skills` | 已保存的数据对比定义/任务 | 迁移数据源从物理表名到数据集标识，并保留历史迁移痕迹 |

#### 4.1.2 新增数据模型

1. **`dataset_acl_audit_logs`**：记录数据集 ACL 与可消费状态的变更审计。

| 字段 | 类型/约束 | 说明 |
| --- | --- | --- |
| `id` | 主键 | 审计记录标识 |
| `dataset_id` | 外键，非空 | 被变更的数据集 |
| `action` | 非空枚举 | `grant_role`、`grant_user`、`revoke_role`、`revoke_user`、`activate`、`deactivate`、`publish`、`unpublish` |
| `principal_type` | 可空枚举 | `role` / `user`；状态变更时为空 |
| `principal_id` | 可空 | 被授权/撤销对象 |
| `before_snapshot` | JSON，可空 | 变更前最小必要授权或状态快照，不保存业务数据 |
| `after_snapshot` | JSON，可空 | 变更后最小必要授权或状态快照，不保存业务数据 |
| `operated_by` | 外键/逻辑用户标识，非空 | 操作者 |
| `created_at` | 非空 | 操作时间 |

索引：`dataset_id, created_at DESC`；`operated_by, created_at DESC`。

2. **`data_consumption_audit_logs`**：记录业务数据消费的最小必要裁决结果。

| 字段 | 类型/约束 | 说明 |
| --- | --- | --- |
| `id` | 主键 | 审计记录标识 |
| `user_id` | 非空 | 发起消费的 Portal 用户 |
| `entry_point` | 非空枚举 | `report`、`dataset_query`、`dataset_export`、`data_compare`、`ai`、`feishu`、`open_api` |
| `operation` | 非空枚举 | `list`、`preview`、`run`、`export`、`compare`、`answer` |
| `dataset_ids` | JSON，非空 | 本次实际参与裁决的数据集 ID 集合；对比可为两个 |
| `decision` | 非空枚举 | `allowed`、`denied_dataset_acl`、`denied_dataset_status`、`denied_scope`、`filtered_fields` |
| `scope_applied` | 非空布尔 | 是否执行了实际限制性范围过滤 |
| `hidden_field_codes` | JSON，非空默认 `[]` | 被隐藏的输出字段代码；不得记录字段值 |
| `trace_id` | 可空、可索引 | 与报表/AI/开放接口追踪关联 |
| `created_at` | 非空 | 发生时间 |

索引：`user_id, created_at DESC`；`entry_point, created_at DESC`；GIN/等效索引 `dataset_ids`（按现有数据库能力选择）。

#### 4.1.3 迁移与兼容

- 新建一条 Alembic migration，创建两张审计表、索引，并为 `dataset_acl` 增加去重约束：同一 `dataset_id + role_id` 或同一 `dataset_id + user_id` 不允许重复。
- migration `upgrade` 前必须清理已有重复 ACL：同一主体保留 ID 最小的一条；清理动作写入 migration 注释和上线说明，不删除数据集本身。
- migration `downgrade` 按依赖顺序删除新增索引、约束与审计表；不得删除已有数据集、数据集 ACL、报表或数据对比历史对象。
- `data_compare_tasks` 和 `ai_skills` 迁移采用兼容期字段：新增 `dataset_a_id`、`dataset_b_id` 与 `dataset_version_snapshot`，保留旧 `table_a/table_b` 只读兼容。后续回填完成并通过验收后，才另行决定删除旧字段。
- 无法唯一映射到单一数据集的历史数据对比技能/任务，必须标记为 `needs_rebind` 并禁用执行，不得猜测映射或自动绑定到任意单表数据集。

### 4.2 后端接口

> 所有业务消费接口均使用当前登录用户；不得接受调用方传入的角色、组织范围、物理表名或未授权字段来扩大权限。

#### 4.2.1 数据集统一授权与发现

| URL | Method | 权限 | 说明 |
| --- | --- | --- | --- |
| `/api/v1/datasets/consumable` | GET | 已登录 + `datasource.datasets:V` 或目标消费入口的读取权限 | 返回当前用户有 ACL、已启用且可消费的数据集摘要 |
| `/api/v1/datasets/{dataset_id}/consumption-schema` | GET | 已登录 + 目标数据集 ACL | 返回消费侧允许的输出字段、筛选字段、关联摘要、范围策略；不返回原始物理表名 |
| `/api/v1/datasets/{dataset_id}/access-summary` | GET | `datasource.datasets:U` 或超级管理员 | 返回 ACL 授权对象、状态、引用报表/对比任务数量和影响摘要 |
| `/api/v1/datasets/{dataset_id}/acl-audit-logs` | GET | `datasource.datasets:U` 或超级管理员 | 分页返回最小必要 ACL 审计记录 |

`GET /datasets/consumable` Response：

```json
{
  "items": [
    {
      "id": 12,
      "code": "ds_employee_movement",
      "label": "人员异动数据集",
      "description": "用于 HRBP 查询组织范围内的人员异动",
      "subject_area": "人力资源",
      "scope_strategy": "person_first",
      "status": "published",
      "has_query_access": true,
      "has_export_access": true
    }
  ]
}
```

约束：只返回已通过 ACL 的数据集；`has_export_access` 仅表示功能导出权限与数据集消费状态的组合摘要，真正导出时仍重新校验。

#### 4.2.2 通用数据集查询与导出

| URL | Method | 权限 | 说明 |
| --- | --- | --- | --- |
| `/api/v1/dataset-queries/{dataset_id}/preview` | POST | 已登录 + `data.view:V` + 目标数据集 ACL | 数据集内受控预览查询 |
| `/api/v1/dataset-queries/{dataset_id}/distinct` | POST | 已登录 + `data.view:V` + 目标数据集 ACL | 获取已批准、可见字段的受控枚举值 |
| `/api/v1/dataset-queries/{dataset_id}/export.csv` | POST | 已登录 + `data.view:E` + 目标数据集 ACL | 导出当前数据集查询结果 |
| `/api/v1/dataset-queries/{dataset_id}/export.xlsx` | POST | 已登录 + `data.view:E` + 目标数据集 ACL | 导出当前数据集查询结果 |

`POST /dataset-queries/{dataset_id}/preview` Request：

```json
{
  "columns": ["employee_name", "org_name", "change_type", "effective_date"],
  "filters": [
    {"field": "effective_date", "operator": "between", "values": ["2026-05-01", "2026-07-31"]}
  ],
  "sort": [{"field": "effective_date", "direction": "desc"}],
  "page": 1,
  "page_size": 50
}
```

Request 规则：

- `columns`、`filters.field`、`sort.field` 仅允许目标数据集已发布的输出字段代码。
- `operator` 使用封闭枚举：`eq`、`in`、`between`、`contains`、`is_null`、`is_not_null`；按字段类型校验 `values`。
- 最多 20 个字段、10 个筛选条件、3 个排序字段；`page_size` 范围 1–200。
- 不允许传入 `table`、`alias`、SQL、JOIN、组织条件、成本中心条件或 scope 条件。

Response：

```json
{
  "dataset_id": 12,
  "columns": [
    {"code": "employee_name", "label": "姓名", "data_type": "string"},
    {"code": "org_name", "label": "所属组织", "data_type": "string"}
  ],
  "items": [{"employee_name": "张三", "org_name": "华东销售部"}],
  "total": 1,
  "page": 1,
  "page_size": 50,
  "permission": {"scope_applied": true, "hidden_columns": []},
  "trace_id": "..."
}
```

状态码：

| 状态码 | 场景 | 对外要求 |
| --- | --- | --- |
| 200 | 查询、空结果、导出成功 | 空结果不得泄露无权限与不存在的差异 |
| 400 | 字段、筛选、排序、分页参数不属于数据集契约或格式非法 | 不回显物理表、SQL 或未授权字段 |
| 401 | 未登录、Token 无效、账号停用 | 统一认证提示 |
| 403 | 无功能权限、无数据集 ACL、无导出权限、数据集不可消费 | 不返回数据集内部表/字段细节 |
| 404 | 数据集不存在或对调用者不可发现 | 对不可发现数据集使用统一不存在反馈 |
| 409 | 数据集结构/状态在请求期间发生不兼容变化 | 提示刷新后重试，不返回部分数据 |
| 429 | 触发查询/导出限速 | 返回统一限速提示 |
| 500 | 未捕获异常 | 返回 `trace_id`，不得返回堆栈、SQL、物理表名 |

#### 4.2.3 数据对比接口迁移

现有 `CompareSpec.source_a.table/source_b.table` 迁移为：

```json
{
  "source_a": {"dataset_id": 12, "period": "202607", "prefilter": [...]},
  "source_b": {"dataset_id": 18, "period": "202607", "prefilter": [...]},
  "join_keys": ["employee_no"],
  "compare_type": "field"
}
```

接口变更：

| URL | Method | 权限 | 改造 |
| --- | --- | --- | --- |
| `/api/v1/data-compare/datasets` | GET | `system.data_compare:V` | 替代 `/tables`；仅返回当前用户可比较的数据集 |
| `/api/v1/data-compare/datasets/{dataset_id}/columns` | GET | `system.data_compare:V` + 数据集 ACL | 替代 `/tables/{table}/columns`；只返回输出字段且执行字段裁决 |
| `/api/v1/data-compare/invoke` | POST | `system.data_compare:V` + 双侧数据集 ACL | CompareSpec 改为数据集标识；两侧分别做 ACL、范围与字段裁决 |
| `/api/v1/data-compare/skills/*` | 既有 | 原功能权限 + 运行时双侧 ACL | 保存/生成/执行时均只接受/解析可授权数据集 |
| `/api/v1/data-compare/tasks/*` | 既有 | 原功能权限 + 执行时双侧 ACL | 历史无映射任务 `needs_rebind`，不允许执行 |

对比执行规则：双侧数据集 ACL 必须在模型调用、字段元数据返回、SQL 编译之前完成；任一侧失败即终止，不返回另一侧的部分结果。

#### 4.2.4 全局 AI 安全数据集上下文

新增内部服务接口（不直接向浏览器暴露原始物理表元数据）：

```text
list_authorized_ai_dataset_context(user, capability) -> AuthorizedDatasetContext[]
```

返回每个数据集的：`dataset_id`、业务名称、业务说明、已批准输出字段的业务名称/类型、允许筛选维度、允许聚合能力、结果行数上限、范围策略摘要。不得返回：物理表名、表别名、原始 SQL、数据库列、未授权数据集、无权字段。

当全局 AI 新增数据问答 Capability 时：

- Capability 必须声明可使用的数据集类别和允许操作；
- 路由/Extractor 只可选择服务端提供的 `dataset_id`；
- Handler 必须调用数据集查询服务，不得调用 `/data/{table}` 或直接使用 `DATA_TABLES`；
- AI 进入模型的上下文和模型得到的结果均为已授权、已字段裁决后的最小必要数据；
- 无可消费数据集时，AI 返回“当前没有可用数据集或无对应权限”，不枚举系统数据资产。

### 4.3 业务逻辑

#### 4.3.1 统一授权服务

新增 `app/datasets/access_service.py`，作为唯一 ACL 判定入口，至少提供：

```text
can_consume_dataset(user, dataset_id, db, purpose) -> AccessDecision
list_consumable_datasets(user, db, purpose) -> list[ConsumableDataset]
get_consumption_schema(user, dataset_id, db, purpose) -> ConsumptionSchema
assert_dataset_fields(user, dataset_id, requested_fields, db, purpose) -> AllowedFields
record_consumption_decision(...) -> None
```

`purpose` 为封闭枚举：`report_view`、`report_run`、`report_export`、`dataset_query`、`dataset_export`、`data_compare`、`ai`、`feishu`、`open_api`。不得由请求客户端任意填写；每个入口在服务端固定传入。

`AccessDecision` 必须区分：

- 数据集不存在；
- 数据集未启用/未发布/不可消费；
- 创建者放行；
- 超级管理员放行；
- 用户 ACL 放行；
- 角色 ACL 放行；
- ACL 拒绝；
- 功能权限拒绝（由入口层先判断）；
- 字段、范围等后续裁决结果不混入 ACL 决策。

安全规则：

1. ACL 判定每次请求查当前有效角色和数据集状态，不将授权结果长期缓存到浏览器、JWT 或 AI 会话。
2. `DataSetAcl` 无记录时仅创建者和超级管理员可消费，保持 fail-closed。
3. 已停用的用户、角色、数据集不可授予或保留有效消费权。
4. 对普通业务用户的“不存在”和“无 ACL”统一为不可发现语义；管理员设计页保留明确诊断。
5. 不允许通过“访问任一包含该表的数据集”获得对该物理表的直接权限。

#### 4.3.2 数据集查询执行器

新增 `app/dataset_query/` 模块，包含：

- `schemas.py`：预览、distinct、导出请求/响应的封闭 Schema；
- `service.py`：授权、字段契约、数据范围、字段裁决、SQL 构建和审计编排；
- `router.py`：仅暴露数据集 ID 查询接口；
- `sql_builder.py`：复用报表现有数据集 SQL 构建能力，禁止拼接调用方原始标识符。

固定执行顺序：

```text
认证/账号启用
→ data.view:V 或 data.view:E 功能权限
→ can_consume_dataset(... purpose=dataset_query/dataset_export)
→ 加载数据集可消费 Schema
→ 请求字段/筛选/排序与 Schema 白名单校验
→ 字段裁决：先移除无权字段及其筛选/排序/聚合使用权
→ build_scope_filter(用户, 数据集范围策略) 在 SQL 编译前注入
→ 构建数据集内允许的表/关联/投影 SQL
→ 执行分页或导出
→ 写入 data_consumption_audit_logs
→ 返回已裁决结果
```

禁止项：

- 不得接受 `table_name`、`alias`、原始列名、SQL、JOIN 条件、scope 条件；
- 不得先全表取数后在 Python 或 AI 层过滤范围；
- 不得在字段隐藏后让该字段参与 distinct、filter、sort、group by、aggregate；
- 不得把报表 SQL 构建器的内部物理表错误直接返回客户端；
- 不得为“方便兼容”在普通用户路径回退至 `/data/{table}`。

#### 4.3.3 报表链路收口

- 将 `reports/router.py` 内部的数据集 ACL 调用迁移至 `access_service`，但保留报表自身可见性/`ReportAcl` 规则。
- 报表列表、详情、运行、复制、CSV/XLSX 导出在每个读取点重新检查目标数据集的消费授权与状态。
- `reports/sql_builder.py` 继续使用数据集输出字段、字段裁决和范围过滤；其调用方必须将目标数据集和 `purpose` 传入统一授权服务。
- 报表创建/编辑页面的数据集下拉仅取 `/datasets/consumable?purpose=report_*` 返回对象。
- 已保存报表的底层数据集被停用、删除或授权撤销时，结果为受控 403/404/409，不得缓存旧结果继续展示或导出。

#### 4.3.4 数据对比链路收口

- `MetadataLoader` 不再面向业务用户加载全部 `registered_tables`；改为 `DatasetMetadataLoader`，仅加载当前用户获双侧授权数据集的消费 Schema。
- `CompareSpec`、`DataSource`、任务与技能持久化对象从物理表字段迁移为数据集 ID；字段引用使用数据集输出字段代码。
- 比较来源为多表数据集时，数据集内部关联仅使用其已保存的关联定义；对比器不可生成额外 JOIN。
- 对比前分别调用两侧 `can_consume_dataset(... purpose=data_compare)`，再分别构建 scope；任一 scope 为拒绝时抛出受控 `ScopeDeniedError`，不得误报“数据一致”。
- 对比结果的敏感字段判定必须从输出字段追溯到统一字段裁决，不再只读取 `TableColumn.is_sensitive`。
- AI 数据对比 extractor 的提示词仅使用授权数据集的业务标签和字段；不得把所有注册表及字段发送给模型。
- 已保存任务在实际执行时再次检查当前执行主体的数据集 ACL；定时任务须保存“执行身份”，缺失或授权失效即失败并记录“需重新授权”，不得以系统全权身份绕过。

#### 4.3.5 历史直接表查询隔离

- `app/data/router.py` 的 `GET /data/{table}`、`/{table}/columns`、`/{table}/distinct`、导出与写操作必须明确区分“管理员维护路径”和“普通业务消费路径”。
- 普通用户入口 `DataView.vue` / `DataTableView.vue` 迁移到数据集查询页面后，不再调用 `/data/{table}`。
- 对保留的底表维护接口增加 `data.raw_maintenance:V/C/U/D/E` 专项权限（仅超级管理员或指定数据管理员角色授予），并要求调用者通过现有数据资产管理权限校验。
- 未具备专项维护权限的登录用户请求直接表读取接口时返回 403；不能只依赖 `current_user`、行级范围或字段隐藏。
- 该隔离避免 URL 猜测与已注册表枚举成为业务数据读取旁路。

#### 4.3.6 审计、事务与异常

- 数据集 ACL 修改与 `dataset_acl_audit_logs` 写入在同一事务提交；失败时 ACL 和审计均回滚。
- 消费审计只记录最小必要元数据：用户、入口、操作、数据集 ID、裁决结果、范围是否生效、被隐藏字段代码、trace ID、时间；不记录行数据、字段原值、原始 SQL、AI 完整对话、未授权字段或组织范围条件。
- 批量导出、数据对比、AI 查询异常时，写入失败审计但不记录敏感输入输出；对外返回统一错误和 `trace_id`。
- 授权服务不能吞掉数据库异常后默认放行；授权、数据集状态、消费 Schema 解析异常均 fail-closed。

### 4.5 权限、安全与外部系统

#### 权限点

| 权限点 | 用途 | 说明 |
| --- | --- | --- |
| `datasource.datasets:V/C/U/D` | 数据集发现、设计、授权配置 | 设计权限不等于数据消费权限；消费仍需数据集 ACL |
| `data.view:V/E` | 通用数据集查询与导出 | 必须与目标数据集 ACL 同时满足 |
| `data.raw_maintenance:V/C/U/D/E` | 受控底表维护 | 新增；只授予超级管理员/数据管理员，不面向普通业务用户 |
| `report.list:V/C/U/D/E` | 报表操作 | 报表操作权限不替代底层数据集 ACL |
| `system.data_compare:V/C/U/D` | 数据对比技能与任务 | 每次生成/执行仍校验双侧数据集 ACL |
| `ai.data_query:V`（新增） | 全局 AI 数据问答 | Capability 还需按服务端可消费数据集上下文执行 |

#### 安全边界

1. **数据集 ACL 是 L2 资源授权，行级范围与字段权限不可被替代或跳过。**
2. **默认拒绝：** 无 ACL、数据集停用、用户/角色停用、无法解析消费 Schema、数据对比任一侧无权时均拒绝。
3. **SQL/动态字段安全：** 所有字段、筛选、排序、关联均由数据集输出 Schema 和封闭枚举验证后映射；不拼接调用方提供的表名、列名、SQL 或 JOIN。
4. **字段安全：** `get_hidden_columns` / `resolve_field_access` 必须在字段选择、filter、distinct、排序、聚合和结果投影的前置阶段执行；不得只在展示层掩码。
5. **范围安全：** `build_scope_filter` 必须在 SQL 执行前注入；多表数据集和数据对比两侧分别应用正确 alias 的范围子句。
6. **AI 安全：** 未授权数据集、物理表结构和原始业务数据不得进入 LLM 提示词；LLM 只能输出封闭查询意图/字段代码，最终执行由确定性服务完成。
7. **外部系统边界：** 飞书 `open_id` 或未来 API 调用方必须映射到有效 Portal 主体，再复用同一 ACL 服务；不得以机器人或系统账号全权读取后回传给用户。
8. **UCP 边界：** UCP/数据连接平台可管理资源、凭证与数据接入，但不成为业务数据消费授权旁路；所有对外业务数据服务仍以数据集 ACL 为边界。

## 7. 验收标准

### 用户验收

1. HRBP 能以业务数据集名称选择并查询获授权主题，且只能看到自己组织/成本中心范围内、字段有权的数据。
2. HRBP 无法通过页面、URL、AI 提问、数据对比、报表共享或导出获取未获授权薪酬主题的数据。
3. 数据管理员能够在数据集编辑页完成角色/用户授权、撤销、状态查看和影响确认，不需要维护物理表 ACL。
4. 资产目录浏览者清楚理解元数据浏览与业务数据查询的区别，且不存在直接读表入口。

### 开发验收

1. 所有已纳入范围的业务数据消费入口均调用统一 `access_service`，不存在复制或绕过的 ACL 判断。
2. 普通业务查询与 AI 查询的输入 Schema 不含物理表名、SQL 或自由 JOIN。
3. 报表、查询、导出、数据对比任一入口均在数据集 ACL、范围和字段权限三层联合通过后才取数。
4. 授权服务和消费执行器遇到异常均 fail-closed。

### 测试验收

1. 第 6 节测试矩阵中所有 P0/P1 场景自动化覆盖；无 ACL、范围外、无字段权限、双侧对比拒绝、授权撤销必须为 P0。
2. 授权一致性：同一用户、同一数据集、同一时点，在报表、通用查询、数据对比、AI 等已接入入口的裁决一致率为 100%。
3. 未授权拦截：验收场景中未授权数据集返回业务记录、字段值、可推断结构的次数为 0。
4. 范围与字段安全：范围外记录返回数为 0；通用消费路径返回无权敏感字段原值次数为 0。

### UI/交互验收

1. 普通用户页面只出现可消费数据集和允许字段，不显示物理表名或未授权对象。
2. 关键状态完整：加载、无数据集、空结果、无权限、数据集停用、字段不可用、保存失败、导出失败、历史对比需重绑。
3. ACL 撤销和数据集停用均有明确确认与影响提示；失败后不丢失管理员未提交编辑内容。
4. 前端生产构建通过，关键路径在目标浏览器中完成 E2E 验证。

### 上线验收

1. 数据库 upgrade/downgrade 演练通过，历史 ACL 和可迁移的历史对比对象数据正确保留。
2. 先以首批低敏数据集和少量角色灰度启用，审计显示授权裁决、范围与字段隐藏信息完整。
3. 发现任一绕过路径时可关闭数据集消费状态或收回 ACL，后续请求立即生效。
4. 上线后首周完成一次未授权访问与高敏主题访问的审计抽查。

## 8. 风险与兼容性

| 风险 | 等级 | 影响 | 应对方案 |
| --- | --- | --- | --- |
| 历史入口遗漏 | 高 | 出现未接数据集 ACL 的数据读取旁路 | X1601 全量盘点；X1612 安全回归；新入口上线门禁 |
| 历史数据对比无法映射数据集 | 高 | 错绑会扩大数据权限 | 标记 `needs_rebind` 并禁用；只允许人工明确重绑 |
| 数据集 ACL 配置不完整 | 高 | 用户突然无法访问或误授权 | 无 ACL fail-closed；灰度矩阵、影响提示、审计与回滚 |
| 数据集输出字段治理不完整 | 高 | 查询/AI 暴露不该消费的字段 | 输出字段是强制白名单；字段分类继续裁决；高敏主题先不接入 |
| 范围过滤与数据集关联别名不一致 | 高 | 范围过滤失效或误拒绝 | SQL 前构建范围、alias 回归测试、多表数据集 P0 测试 |
| 报表/导出历史兼容 | 中 | 已保存报表在授权收口后无法运行 | 提前盘点引用；明确受控拒绝提示；管理员修复 ACL/字段 |
| AI 将表名或字段名注入上下文 | 高 | 可能产生越权尝试或信息泄露 | AI Context Builder 只提供授权数据集业务 Schema；禁止物理表/SQL |
| 资产目录被误认为数据入口 | 中 | 用户预期与安全策略冲突 | 页面提示、仅展示已授权数据集链接、移除直接读表跳转 |
| 授权服务性能开销 | 中 | 列表、报表/AI 查询变慢 | 服务端批量查询 ACL 与 Schema；只缓存请求级元数据；权限变更不使用长期缓存 |
| 审计记录过度收集 | 中 | 产生敏感数据二次暴露 | 最小字段审计；禁止保存结果行、字段值、SQL、完整 AI 对话 |
| 飞书/开放 API 使用系统身份 | 高 | 外部渠道绕过最终用户权限 | 强制映射 Portal 主体；guard 拒绝匿名/系统全权读取 |

## 10. 假设与可配置项

### 假设

1. 数据集继续作为唯一面向普通业务用户的最小可授权数据产品；物理表仅用于管理员接入、建模、治理和受控维护。
2. 普通业务用户的数据范围继续复用现有组织树、成本中心树、人员条件与 `scope_strategy`，本期不改团队口径。
3. 超级管理员可访问全部数据集，但其消费行为仍记录审计，且字段合规策略如有组织另行规定则优先。
4. 数据对比任务的定时执行必须关联实际执行用户；不允许无用户上下文的系统全权执行。

### 可配置项

以下规则必须在本期功能中实现为管理员可维护或系统可配置的能力；**不要求在开发前确定首批具体取值**。未配置时采用安全默认值，后续通过配置生效，不需要修改代码或重新部署。

| 配置项 | 管理对象与默认值 | 配置目的 | 生效规则 |
| --- | --- | --- | --- |
| 数据集授权矩阵 | 每个数据集的角色 ACL、用户 ACL；默认无 ACL | 控制哪些角色/用户可消费每个业务数据集 | 无 ACL 时仅创建者与超级管理员可消费；授权变更在下一请求生效 |
| 高敏主题策略 | 数据集 ACL + 消费状态；默认不向普通角色授权 | 让薪酬、绩效、劳动关系、医疗健康等主题按需逐项开放 | 未配置 ACL 或未发布时不可被业务查询、AI、对比或导出消费 |
| 底表维护角色范围 | `data.raw_maintenance:V/C/U/D/E` 的角色授权；默认仅超级管理员 | 将数据治理/排障与普通业务消费隔离 | 未获专项维护权限者直接访问 `/data/{table}` 返回 403 |
| 数据集消费状态 | 数据集 `is_active`、`status` 与可消费规则；默认仅 `is_active=true 且 status=published` | 控制数据集从设计、试运行到正式消费的生命周期 | 非可消费状态对普通用户不可发现、不可查询、不可导出、不可被 AI/对比使用 |
| 内部试运行名单 | 指定数据集的角色/用户 ACL；默认空 | 允许在正式发布前仅向指定试运行人员开放 | 仅当数据集处于允许试运行状态且 ACL 命中时可消费；其他用户仍不可发现 |
| 审计保留期 | ACL 变更审计默认保留 2 年；数据消费审计默认保留 1 年 | 满足授权追溯、问题排查与数据治理 | 管理员可调整保留期；到期后按配置归档或清理，且不保存业务行数据、字段值、SQL 或完整 AI 对话 |
| 审计查看与导出范围 | 审计角色 ACL；默认超级管理员与数据管理员可查看，默认禁止导出 | 防止审计系统成为二次敏感数据泄露入口 | 未授权者不可查看审计明细；导出需单独授予审计导出权限 |

> 配置管理界面与权限点属于本期范围。首批角色、首批数据集和具体保留期属于上线运营配置，不阻塞开发。
