# 飞书多维表格可复用连接与数据对象配置

## 1. 背景与目标

- 背景：当前 UCP 的“接入系统 → 资源”模型将资源定义为一张数据表或一个外部 API。对飞书多维表格而言，如果每张业务表都创建资源，会造成大量重复的凭证、适配器和连接配置；同时当前代码仅提供飞书招聘 Offer 查询适配器，尚不能读取飞书多维表格。
- 现状：`UcpResource` 已支持适配器、凭证与动态 Schema 配置；流水线设计器可选择资源并在步骤中附加 `params`。但执行引擎仍以旧 `UcpSystemConfig` 为优先执行对象，未完整消费新资源的 `resource_id`；也没有“一个飞书多维表格资源下管理多张表”的数据对象实体和 UI。
- 目标：提供一个可复用的“飞书多维表格读取”资源。管理员只需配置一次飞书应用凭证与通用资源，即可在该资源下创建多张表的数据对象配置，并在多个流水线步骤中选择其中一张表执行读取。
- 非目标 / 不做范围：
  - 不自动发现或批量导入企业所有飞书多维表格。
  - 不在代码或数据库明文保存飞书 `app_secret`、临时 token 或用户 token。
  - 不实现飞书电子表格（Sheets）、飞书云文档、写入多维表格、双向同步、全量数据仓库落库编排。
  - 不替换既有的北森、招聘 Offer、滴滴或历史 `UcpSystemConfig` 流程。

### 假设与待确认事项

- 飞书开放平台应用已由管理员创建，并在上线前获批读取多维表格记录的权限；本期仅提供系统配置和调用能力。
- 资源凭证采用现有 `UcpCredential.secrets_encrypted`，约定 `app_id` / `app_secret`，也兼容已配置的 `tenant_access_token`。
- 每个数据对象配置对应一张飞书多维表格中的一个数据表（`app_token + table_id`）；`view_id`、字段映射、筛选条件为可选配置。

## 2. 用户场景

### 2.1 UCP 管理员：一次建立飞书连接

- 入口：数据连接 → 接入系统 → 添加系统。
- 操作：创建“飞书”系统；创建飞书应用凭证；创建资源并选择“飞书多维表格读取”。
- 系统反馈：资源表单仅要求通用连接配置；不会要求填写某一张表的 `app_token` 或 `table_id`。
- 成功结果：一个资源可被多条数据对象配置和多个流水线复用。
- 失败/空态/无权限：未选择凭证、凭证不属于当前系统、适配器未启用时禁止保存；无查看权限时不显示数据对象配置入口。

### 2.2 UCP 管理员：在资源下新增多张多维表格

- 入口：打开“飞书多维表格读取”资源详情 → 数据对象配置。
- 操作：新增对象编码、对象名称、`app_token`、`table_id`、可选 `view_id`、字段映射和读取限制。
- 系统反馈：校验同一资源内对象编码唯一、`app_token` / `table_id` 必填、读取上限在安全范围内；保存后显示启用状态和最近更新人。
- 成功结果：例如“员工花名册”“候选人库”“培训记录”都挂在同一个飞书资源下。
- 失败/空态/无权限：资源不使用飞书多维表格适配器时，不展示该区域；空态引导“新增数据对象”；无编辑权限只读。

### 2.3 流水线配置人员：选择要读取的具体表

- 入口：数据连接 → 流水线设计器 → 资源调用节点。
- 操作：先选择飞书系统和通用资源，再选择该资源下已启用的数据对象。
- 系统反馈：节点展示“资源 / 数据对象”；选择资源后异步加载对象列表；未选对象时飞书多维表格节点标记为配置不完整。
- 成功结果：同一资源能被多个节点复用，每个节点读取不同表；节点级 `params` 仅用于临时非敏感覆盖，不覆盖凭证。
- 失败/空态/无权限：对象被停用、删除或不属于资源时，保存/执行返回明确错误；无查看权限不返回对象名称与定位信息。

### 2.4 执行人员：运行和排障

- 入口：流水线手工触发或计划执行。
- 系统反馈：执行引擎用资源凭证换取飞书访问令牌，按对象配置分页拉取记录，写入既有执行日志和资源数据快照。
- 成功结果：结果包含标准化记录数组、行数和分页信息；字段映射启用时输出平台字段名。
- 失败：凭证无效、权限不足、表不存在、对象停用、飞书 API 返回异常时返回可识别错误，不泄露密钥或 token。

## 3. 功能范围

| 功能项 | 是否本期实现 | 说明 |
|---|---:|---|
| 飞书多维表格读取适配器 | 是 | 新增 `FEISHU_BITABLE_PULL_ADAPTER`，读取一张已配置的数据表。 |
| 内置适配器元数据与动态 Schema | 是 | 启动时自动登记、启用，资源页可选择。 |
| 多维表格数据对象配置 CRUD | 是 | 资源下新增/查询/更新/启停/删除配置。 |
| 流水线按资源 + 数据对象执行 | 是 | 新资源模型优先；旧 `UcpSystemConfig` 保持兼容。 |
| 流水线设计器对象选择控件 | 是 | 仅飞书多维表格资源显示。 |
| 资源详情数据对象管理 UI | 是 | 列表、空态、新增、编辑、启停、删除。 |
| 飞书电子表格（Sheets） | 否 | 与多维表格 API、定位标识不同，后续单独适配。 |
| 写入/更新飞书多维表格 | 否 | 本期只读。 |
| 自动同步/爬取企业内全部表 | 否 | 防止权限过大和误接入。 |

## 4. 技术设计

### 4.1 数据库 / 数据模型

新增表：`ucp_bitable_table_config`。

| 字段 | 类型 / 默认值 | 说明 |
|---|---|---|
| `id` | BigInteger PK | 主键。 |
| `resource_id` | FK → `ucp_resource.id`，非空 | 所属通用飞书多维表格资源，删除资源时级联删除。 |
| `object_code` | String(64)，非空 | 同资源内唯一、使用大写字母数字下划线。 |
| `object_name` | String(128)，非空 | 业务展示名称。 |
| `app_token` | String(128)，非空 | 多维表格应用标识，非密钥。 |
| `table_id` | String(128)，非空 | 数据表标识，非密钥。 |
| `view_id` | String(128)，可空 | 可选视图筛选。 |
| `field_mapping` | JSON，默认 `{}` | 源字段名 → 输出字段名；不填时保留飞书原字段名。 |
| `filter_config` | JSON，默认 `{}` | 预留筛选、字段选择等非敏感参数。 |
| `page_size` | Integer，默认 `100` | 单页读取记录数，限制 1–500。 |
| `max_records` | Integer，默认 `10000` | 单次最多读取记录数，限制 1–50000。 |
| `is_active` | SmallInteger，默认 `1` | 是否可被流水线选择和执行。 |
| 审计字段 | created/updated by/at | 与 UCP 资源一致。 |

- 唯一索引：`(resource_id, object_code)`。
- 查询索引：`(resource_id, is_active)`。
- Migration：新增 `0113_ucp_feishu_bitable_resource.py`，`upgrade` 建表建索引，`downgrade` 按索引、表逆序删除。
- 兼容：不修改旧表字段、旧资源或旧流水线；无 `bitable_table_id` 的非飞书资源继续走旧逻辑。

### 4.2 后端接口

所有接口挂在 `/api/v1/ucp`，遵循既有 `require_op("ucp.systems", ...)` 权限模式。

| URL | Method | 权限 | 请求 / 返回 | 状态与错误 |
|---|---|---|---|---|
| `/resources/{resource_id}/bitable-tables` | GET | `ucp.systems:V` | `is_active?`；返回 `{total, items}` | 404 资源不存在；400 资源不是飞书多维表格资源。 |
| `/resources/{resource_id}/bitable-tables` | POST | `ucp.systems:C` | 对象编码、名称、app_token、table_id、view_id、映射、分页限制 | 201；400 参数/资源/凭证错误；409 同编码。 |
| `/resources/{resource_id}/bitable-tables/{id}` | PATCH | `ucp.systems:U` | 可更新业务字段与启用状态 | 404 不存在或不归属资源；400 校验失败。 |
| `/resources/{resource_id}/bitable-tables/{id}` | DELETE | `ucp.systems:D` | 返回 `{deleted: id}` | 404 不存在；被流水线引用时 409。 |
| `/resources/{resource_id}/bitable-tables/{id}/preview` | POST | `ucp.systems:V` | `{limit?: 1..100}`；返回标准 AdapterResult | 400 资源/对象不匹配；502 外部系统失败。 |

接口不返回凭证明文、飞书 access token 或 URL 中的认证信息。

### 4.3 业务逻辑

1. `FEISHU_BITABLE_PULL_ADAPTER`：
   - 优先使用凭证中的 `tenant_access_token`；没有时使用 `app_id` / `app_secret` 获取 tenant access token。
   - 调用飞书多维表格记录读取接口并按 `page_token` 分页，直到无更多数据或达到 `max_records`。
   - 请求参数来自“资源协议配置 + 数据对象配置 + 流水线非敏感 params”；数据对象的 `app_token`、`table_id` 不允许被非管理员运行参数覆盖。
   - 返回 `AdapterResult`：`data` 为记录字段对象；`extra` 包含 `app_token/table_id` 的非敏感标识、分页数和是否截断。
   - 飞书字段值保持 JSON 兼容；按 `field_mapping` 重命名顶层字段。
2. 数据对象服务：
   - 仅允许 `resource.adapter_code == FEISHU_BITABLE_PULL_ADAPTER` 的资源维护对象。
   - 校验对象编码、页大小、最大记录数、字段映射均为安全 JSON 对象。
   - 删除前扫描新版流水线步骤 `config.bitable_table_id`，被引用时禁止删除。
3. 流水线执行兼容：
   - `resource_id` 存在时优先加载 `UcpResource`，解密该资源的凭证，并合并 `protocol` / `report_config` / 节点 `params`。
   - 飞书多维表格节点必须提供 `bitable_table_id`，服务端校验对象归属、启用及资源适配器匹配后合入对象配置。
   - 不存在 `resource_id` 时维持旧 `UcpSystemConfig` 执行路径，避免回归旧流程。
4. 禁止重复建设：继续复用 `AdapterResult`、`decrypt_credential_secrets`、现有执行日志、熔断、限流、资源快照、动态 Schema、`UcpCredential` 加密存储。

### 4.4 前端与 UI/交互

1. 接入系统资源页：
   - 页面：`/ucp/systems` 中的系统详情资源抽屉。
   - 选择“飞书多维表格读取”后，显示通用协议字段和“数据对象配置”区域。
   - 数据对象列表字段：名称、编码、App Token（中间脱敏）、Table ID（中间脱敏）、View ID、状态、更新人、更新时间、操作。
   - 空态：说明“一个资源可配置多张飞书多维表格”，提供“新增数据对象”按钮。
   - 新增/编辑弹窗：对象编码、对象名称、App Token、Table ID、View ID、字段映射 JSON、单页条数、最大记录数、启用开关；JSON 非法时前端阻止提交并显示字段级错误。
   - 支持“预览前 100 条”按钮，加载态禁用重复点击，成功显示行数及前若干条脱敏/截断 JSON，失败显示后端安全错误。
2. 流水线设计器：
   - 资源调用节点选择到飞书多维表格资源后出现“数据对象”下拉框；选项为已启用对象。
   - 节点摘要显示资源名称和对象名称；未选对象标记为配置错误并禁止保存/试运行。
   - 切换资源时清空不属于新资源的数据对象。
3. 空态、错误态、权限态：严格复用 Element Plus `ElMessage`、`ElEmpty`、`ElSkeleton` 风格；无权限不渲染写操作，接口 403 显示“无操作权限”。

### 4.5 权限、安全与外部系统

- 凭证仅通过现有加密服务存储，列表与日志只展示脱敏摘要；`tenant_access_token` 不落执行日志。
- `app_token`、`table_id` 不是密钥，但 UI 仍做中间脱敏，防止在截图和公共屏幕暴露内部资源定位信息。
- 所有外部 HTTP 请求固定到飞书开放平台基础地址，不接受资源/对象配置中的任意 URL，沿用/遵守现有 SSRF 边界。
- 页面和接口均校验资源与对象所属关系，禁止跨资源 ID 枚举读取。
- 映射字段仅是字典键重命名，不执行表达式、模板、SQL 或动态代码。
- 上线前由飞书管理员配置应用权限、可访问的数据范围和真实凭证；本期不写入任何生产凭证。

## 6. 测试计划

- 成功：创建飞书资源、创建多张对象配置、流水线分别读取两张表、映射字段正确。
- 参数错误：非法对象编码、空 `app_token` / `table_id`、错误 JSON、超出分页限制。
- 资源不存在：资源或对象不存在、对象不属于资源、非飞书适配器资源访问对象接口。
- 权限不足：按查看/创建/更新/删除权限验证 403；页面不显示越权操作。
- 空数据：飞书返回空 records，执行成功且行数为 0。
- 边界值：分页 1、500；最大记录 1、50000；多页与截断。
- 兼容：旧 `UcpSystemConfig` 流水线仍可执行；非飞书资源 UI 不显示对象配置。
- 安全：日志/接口不含 `app_secret` 或 access token，外部 URL 不可配置。
- 构建：`pytest` 定向、前端生产构建、Docker Compose 开发环境重建、健康检查。

## 7. 验收标准

## 8. 风险与兼容性

| 风险 | 等级 | 影响 | 应对方案 |
|---|---|---|---|
| 飞书应用未获读取权限 | 高 | 请求 403，无法读取表 | 上线检查权限和应用可见范围；返回明确权限错误。 |
| 多维表字段类型复杂 | 中 | 字段值结构不统一 | 保持 JSON 值；仅做顶层字段重命名。 |

| 新旧资源模型并存 | 高 | 旧流水线回归风险 | 执行器以 `resource_id` 分支，保留旧 `UcpSystemConfig` 路径并回归测试。 |

| 内部表定位信息泄露 | 低 | 暴露业务结构 | UI 脱敏，接口按权限返回，日志不记录敏感凭证。 |
