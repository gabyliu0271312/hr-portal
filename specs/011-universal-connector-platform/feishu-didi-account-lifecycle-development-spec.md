 # 飞书员工事件驱动的企业滴滴账号生命周期管理

## 1. 背景与目标

- 背景：HR Portal 已有 UCP 事件总线、流水线、凭证、外部账号生命周期及审计基础能力，前端已有“外部账号生命周期”入口。
- 现状：飞书 Webhook 已可接收、验签/解密并标准化部分事件；滴滴账号适配器已声明 `CREATE`、`UPDATE`、`DISABLE`、`REACTIVATE`、`DELETE` 动作。
- 现状缺口：滴滴客户端当前返回 `simulated=true`，未调用企业滴滴正式接口；运行数据库没有滴滴资源、飞书事件触发器、实际运行流水线；“入职账号创建 / 离职账号停用”只是画布模板，不能直接执行。
- 目标：建设“底层可复用组件 + 配置化规则”，通过规则驱动飞书员工入职、离职、信息变更到企业滴滴账号的创建、更新、停用、恢复、删除。
- 目标：使曹操、门禁、VPN、邮箱等后续系统可复用员工事件、账号映射、审批、延时、审计、通知与重试能力。
- 原则：入职创建或恢复账号；离职默认立即停用，保留期结束且符合企业规则后再删除；不将“离职删除滴滴”实现为专用硬编码功能。
- 非目标 / 不做范围：本期不接入飞书审批实例作为离职事实来源，不修改滴滴行程、账单、结算业务，不重构无关 UCP 模块；未取得滴滴正式接口资料前不允许上线真实删除。

### 假设与待确认事项

1. 飞书租户可订阅反映入职、离职、用户更新的事件，管理员可授予自建应用必要权限。
2. 企业滴滴提供成员查询、创建/导入、停用/禁用、删除或注销接口；最终 URL、认证、签名、字段、错误码以供应商正式资料为准。
3. `employee_id` 为稳定的人事唯一键；手机号可变，不能作为账号唯一关联键。
4. 默认保留期为 30 天；需由 HR、行政/差旅、信息安全共同确认。滴滴不支持物理删除时终态为 `DISABLED`。
5. 以 `ExternalAccount.external_user_id` 作为滴滴动作目标；新建/恢复账号必须持久化该映射。历史账号盘点、导入和人工待处理不作为当前上线前置，映射不确定时不得自动删除。

## 2. 用户场景

### 2.1 UCP 管理员：配置企业滴滴连接器

- 入口：`UCP → 系统管理 / 凭证管理 / 资源管理`。
- 操作：创建业务系统 `DIDI`、生产凭证、账号生命周期资源，执行连通性测试。
- 系统反馈：校验资源编码、适配器、凭证完整性；结果仅展示脱敏地址和响应摘要。
- 成功结果：资源可被规则和流水线选择。
- 失败/空态/无权限：无凭证不可启用；认证、权限、超时、限流写测试日志；无 `ucp.systems`、`ucp.credentials`、`ucp.resources` 权限时只读或接口返回 403。

### 2.2 UCP 管理员：配置生命周期规则

- 入口：`UCP → 事件触发器 / 外部账号生命周期`。
- 操作：选择飞书事件、过滤条件、字段映射、滴滴资源、动作、审批、保留期、通知和失败策略。
- 系统反馈：启用前校验员工唯一键映射、资源、删除策略、审批/保留期；提供不调用滴滴的 dry-run。
- 成功结果：规则启用后生成回调 URL，飞书事件可自动执行。
- 失败/空态/无权限：无飞书连接器显示引导空态；事件、字段、资源无效则禁止启用；无 `ucp.triggers` / `ucp.external_accounts` 权限则禁止操作。

### 2.3 员工入职与离职

- 入职：飞书事件进入后，系统映射员工信息、查询账号映射；无账号则创建，已停用则按规则恢复，已启用则幂等成功。
- 离职：按 `employee_id` 找账号，默认停用；审批完成且保留期到达后删除。
- 系统反馈：事件、任务、流水线、外部账号审计可按追踪 ID 查看状态。
- 异常：账号不存在记录 `ACCOUNT_NOT_FOUND` 而不调用删除；重复事件不重复操作；未结算事项、接口拒绝或删除不支持时保持停用并告警；重新入职按状态机恢复或重新创建。

## 4. 技术设计

### 4.1 数据库 / 数据模型

#### 复用现有模型

- `ucp_event`：标准化事件、原始载荷、状态和追踪 ID。
- `ucp_event_trigger`：事件来源、类型、目标流水线、飞书校验配置、回调路径。
- `ucp_pipeline_config`：运行时步骤、触发方式、启停、错误策略。
- `ucp_system`、`ucp_system_config`、`ucp_credentials`：系统、资源和加密凭证。
- `external_accounts`、`external_account_audits`：账号状态、外部账号 ID 和审计。

#### 新增表：`ucp_account_lifecycle_rule`

| 字段 | 类型 / 默认值 | 说明 |
|---|---|---|
| `id` | bigint / 自增 | 主键。 |
| `rule_code` | varchar(64) | 唯一编码，如 `FEISHU_DIDI_OFFBOARD`。 |
| `rule_name` | varchar(128) | 规则名称。 |
| `source_system_code` | varchar(64) / `FEISHU` | 事件来源。 |
| `feishu_event_type` / `internal_event_type` | varchar(128) | 原始事件和 UCP 标准事件。 |
| `target_system_code` / `target_resource_code` | varchar(64) | 如 `DIDI` / `DIDI_ACCOUNT`。 |
| `lifecycle_action` | varchar(16) | `CREATE/UPDATE/DISABLE/REACTIVATE/DELETE`。 |
| `filter_rule` / `field_mapping` | json / `{}` | 过滤与字段映射；必须包含 `employee_id`。 |
| `account_match_strategy` | varchar(32) / `EMPLOYEE_ID` | 本期仅支持员工编号。 |
| `approval_required` | smallint / `0` | 是否审批。 |
| `retention_days` | integer / `0` | 延时天数。 |
| `failure_policy` / `notification_config` | varchar / json | 失败处理和通知。 |
| `status` | smallint / `0` | `0=禁用`，`1=启用`。 |
| 审计字段 | 用户 / 时间 | 创建、更新人员与时间。 |

#### 新增表：`ucp_account_lifecycle_job`

用于可靠执行延时删除，禁止只依赖 `asyncio.create_task`。

| 字段 | 类型 / 默认值 | 说明 |
|---|---|---|
| `id` / `job_code` | bigint / varchar(64) | 主键和唯一任务编码。 |
| `rule_id` / `account_id` / `event_id` | bigint | 关联规则、账号、来源事件。 |
| `action` / `status` | varchar | 动作及 `PENDING/RUNNING/WAITING_APPROVAL/SUCCESS/FAILED/CANCELLED` 状态。 |
| `scheduled_at` / `executed_at` | timestamptz | 计划和实际执行时间。 |
| `idempotency_key` | varchar(128) | 唯一；建议规则、账号、动作、生效日组合。 |
| `retry_count` / 错误字段 | integer / varchar / text | 重试与最近错误。 |
| `payload_snapshot` | json / `{}` | 脱敏输入快照。 |

#### 索引、兼容、Migration 与回滚

- 添加规则编码、任务幂等键唯一索引，以及任务状态与计划时间复合索引。
- 对外部账号添加 `(system_code, employee_id)` 唯一约束/索引前，迁移前检查必须识别历史一人多账号/缺员工编号数据；冲突数据不自动修复、不自动导入，且不得启用删除动作。
- Alembic migration 必须包含 upgrade/downgrade；downgrade 只删除新增表、索引、外键，绝不删除已有事件、账号、审计。
- 当前仅对新建、查询确认或已存在映射的账号执行生命周期动作；历史账号批量盘点、导入和人工待处理清单延期，无法确认映射时只记录安全告警并禁止自动删除。

### 4.2 后端接口

| URL | Method | 权限 | Request Schema | 响应 / 状态码 |
|---|---|---|---|---|
| `/api/v1/ucp/account-lifecycle-rules` | GET | `ucp.lifecycle_rules:V` | 来源、目标、状态、分页 | `200`：列表和 total。 |
| `/api/v1/ucp/account-lifecycle-rules` | POST | `ucp.lifecycle_rules:C` | 编码、事件、资源、动作、映射、策略 | `201`；`422` 校验失败；`409` 编码重复。 |
| `/api/v1/ucp/account-lifecycle-rules/{rule_code}` | GET | `ucp.lifecycle_rules:V` | 无 | `200`；`404` 不存在。 |
| `/api/v1/ucp/account-lifecycle-rules/{rule_code}` | PUT | `ucp.lifecycle_rules:U` | 可更新配置，不改编码 | `200`；启用中破坏性变更 `409`。 |
| `/api/v1/ucp/account-lifecycle-rules/{rule_code}/enable` | POST | `ucp.lifecycle_rules:U` | `{}` | `200`；配置不完整 `422`。 |
| `/api/v1/ucp/account-lifecycle-rules/{rule_code}/disable` | POST | `ucp.lifecycle_rules:U` | `{}` | `200`；停止新任务。 |
| `/api/v1/ucp/account-lifecycle-rules/{rule_code}/dry-run` | POST | `ucp.lifecycle_rules:V` | 飞书样例事件 | `200`：映射、命中账号、拟执行动作；绝不调用滴滴。 |
| `/api/v1/ucp/account-lifecycle-jobs` | GET | `ucp.lifecycle_jobs:V` | 规则、状态、账号、分页 | `200`：任务列表。 |
| `/api/v1/ucp/account-lifecycle-jobs/{job_code}/retry` | POST | `ucp.lifecycle_jobs:U` | `reason?` | `200`：已重试；`409`：不可重试。 |
| `/api/v1/ucp/external-accounts/{account_id}/reconcile` | POST | `ucp.external_accounts:U` | `{}` | `200`：查询滴滴后回写；外部异常 `502`。 |
| `/api/v1/ucp/external-accounts/import/didi` | POST | `ucp.external_accounts:C` | 导入参数或任务引用 | `202`：盘点任务已创建。 |

- Webhook 继续使用 `POST /api/v1/ucp/webhooks/feishu/{trigger_code}`；URL 校验快速返回 challenge，普通事件接收成功后返回 `200`，真实执行异步。
- 校验失败返回 `401`，触发器不存在 `404`，载荷不能解析 `422`，服务不可用 `503`。
- 接口、日志、前端不得回传完整手机号、密钥、Authorization、签名原文或完整第三方响应。

### 4.3 业务逻辑

#### 组件分层与禁止重复建设

1. **飞书事件连接器**：接收、校验、解密、识别、标准化、幂等入库；滴滴业务不得再次实现 Webhook。
2. **员工事件规范化组件**：将飞书事件转换为 `EMPLOYEE_ONBOARD`、`EMPLOYEE_OFFBOARD`、`EMPLOYEE_UPDATED`。
3. **外部账号生命周期服务**：负责查找、幂等、状态机、审计、失败记录；供应商适配器不得直接更新数据库状态。
4. **企业滴滴连接器**：仅处理滴滴认证、签名、请求、响应、错误码；统一实现 `create/update/disable/reactivate/delete/query`。
5. **规则和任务调度组件**：处理映射、过滤、审批、延时、通知、重试；不得将保留期硬编码至滴滴客户端。

#### 入职流程

1. 飞书事件验证后写入 `ucp_event`，以 `event_id` 幂等。
2. 标准化为 `EMPLOYEE_ONBOARD`，匹配启用的入职规则。
3. 按配置映射 `employee_id`、姓名、手机号、部门；缺关键字段时失败且不调用滴滴。
4. 按 `DIDI + employee_id` 查找账号：不存在则创建，`DISABLED` 按规则恢复，`ACTIVE` 幂等成功。
5. 保存滴滴返回的外部账号 ID、状态和脱敏审计。

#### 离职流程

1. 飞书离职事件标准化为 `EMPLOYEE_OFFBOARD`；原始事件类型必须依据真实联调样例确认。
2. 用 `employee_id` 查找滴滴账号；未找到只记录 `ACCOUNT_NOT_FOUND` 和告警。
3. 默认执行 `DISABLE`；已停用或删除时幂等返回。
4. 配置删除时创建持久化延时任务；需要审批时进入 `WAITING_APPROVAL`。
5. 到期后删除前再次查询滴滴：存在未结算事项或删除受限时保持停用并记录原因。
6. 成功置为 `DELETED`；失败退避重试，达上限后告警并进入人工队列。

#### 状态机与边界

- 合法状态转换：`PENDING → ACTIVE`、`ACTIVE → DISABLED`、`DISABLED → ACTIVE`、`DISABLED → DELETED`。
- 禁止 `DELETED → ACTIVE`；重新入职走创建流程并绑定新的外部账号 ID。
- `DELETE` 不得跳过账号定位、停用校验和必要审批；手工删除也必须复用同一服务和审计。
- 运行时流水线通过 `resource_code` 读取资源，适配器使用注册值 `DIDI_ACCOUNT_PUSH_ADAPTER`，不得使用模板展示名称。

### 4.4 前端与 UI/交互

| 页面 | 路径 | 说明 |
|---|---|---|
| 生命周期规则列表 | `/ucp/account-lifecycle-rules` | 查看、筛选、创建、启停规则。 |
| 规则编辑页/抽屉 | `/ucp/account-lifecycle-rules/:ruleCode` | 配置事件、映射、动作、审批、通知、dry-run。 |
| 外部账号详情扩展 | 现有 `/ucp/external-accounts` | 展示来源规则、任务、事件和审计。 |
| 生命周期任务列表 | `/ucp/account-lifecycle-jobs` | 查看等待审批、延时、失败和重试任务。 |

- 编辑页按“基本信息 → 事件来源 → 字段映射 → 账号动作 → 审批与保留 → 通知与重试 → 测试与启用”分区。
- 字段包含：规则编码、飞书事件类型、内部事件、过滤条件、`employee_id/employee_name/employee_mobile/department` 映射、目标资源、动作、保留天数、审批、通知。
- `DELETE` 展示高风险警告；强制配置保留天数和审批策略。所有交互按钮使用 `PermissionButton`。
- 必须覆盖加载、空态、字段错误、接口错误、无权限、保存成功、测试成功；测试结果仅展示脱敏 JSON。
- 账号详情增加“生命周期”区域：状态、规则、最近事件、计划删除时间、最近错误和审计。任务列表展示任务、员工、动作、规则、计划时间、状态、重试、错误和操作。

### 4.5 权限、安全与外部系统

- 新增 `ucp.lifecycle_rules`（`V/C/U/D`）及 `ucp.lifecycle_jobs`（`V/U`）权限；复用 `ucp.external_accounts`。
- `DELETE` 为高风险操作：记录操作人、事件 ID、规则版本、审批 ID、脱敏请求/响应摘要。
- 凭证仅保存在 `ucp_credentials.secrets_encrypted`；前端只能展示是否配置、有效期和脱敏元数据。
- 所有滴滴请求统一通过 `DidiAccountClient`；禁止业务服务绕过连接器自行拼接 URL 或请求。
- `base_url` 必须使用允许列表；生产/测试用独立资源与凭证；超时、TLS、限流、熔断、退避放在资源配置层。
- 映射仅允许读取白名单 JSONPath，不允许表达式执行、模板代码或动态 SQL；导入任务做格式、大小、列白名单校验。

## 6. 测试计划

| 类别 | 场景 | 预期结果 |
|---|---|---|
| 成功 | 入职事件且无账号 | 创建账号、保存外部账号 ID、状态 `ACTIVE`、审计成功。 |
| 成功 | 离职事件且账号 `ACTIVE` | 停用账号、创建延时任务、审计完整。 |
| 成功 | 到期且审批通过 | 删除账号并置 `DELETED`。 |
| 参数错误 | 缺员工编号、姓名、手机号 | 不调用滴滴，给出字段错误。 |
| 资源错误 | 规则资源/凭证停用 | 禁止启用或执行失败，错误可定位。 |
| 权限 | 无权限用户调用 UI/API | UI 不展示操作，API 返回 403。 |
| 空数据 | 离职未找到账号 | 记录 `ACCOUNT_NOT_FOUND`，不删除其他账号。 |
| 幂等 | 相同 event_id 重复投递 | 不重复创建、停用、删除或创建任务。 |
| 边界 | 保留期 0、最大值、停用后重入职 | 策略校验正确，状态机正确。 |
| 外部异常 | 鉴权、4xx/5xx、超时、限流、响应异常 | 错误归一化、按策略重试、无敏感泄漏。 |
| 兼容 | 历史账号无编号或一人多账号 | 不触发自动删除；记录安全告警，不在本期生成盘点导入或人工待处理清单。 |
| 安全 | 非法签名、篡改映射、非法 base_url | 拒绝并写安全日志。 |
| 回归 | 后端、前端构建及现有 UCP 功能 | 均通过，不影响北森、飞书招聘、曹操。 |

## 7. 验收标准

### 用户验收

- 管理员不修改代码即可配置飞书员工事件到滴滴账号动作。
- 可查看账号状态、规则、事件、任务、失败原因与审计。
- 默认离职停用；事件重复、匹配失败、外部异常均不会误删。

### 开发验收

- 滴滴连接器不再以 `simulated=true` 伪造生产成功。
- 规则、任务、事件、账号、审计可用追踪 ID 关联。
- 延时删除持久化，服务重启后可恢复。

### 测试验收

- 第 6 节场景均有自动化或可复现手工证据。
- migration upgrade/downgrade、后端测试、前端构建、回归通过。
- 滴滴测试环境完成创建、停用、恢复、删除（如支持）联调。

### UI/交互验收

- 规则和任务页面具有加载、空态、错误、无权限、高风险确认。
- 敏感字段、密钥、第三方响应均脱敏。
- 新增菜单与按钮权限符合项目规范。

### 上线验收

- 先白名单灰度，删除在停用稳定后独立启用。
- 已配置事件接收失败、滴滴认证失败、连续任务失败、任务积压、删除拒绝告警。
- 可执行停止规则、停止调度、撤销凭证和人工恢复账号的回滚手册。

## 8. 风险与兼容性

| 风险 | 等级 | 影响 | 应对方案 |
|---|---|---|---|
| 滴滴限制/不支持删除 | 高 | 无法物理删除 | 终态设为 `DISABLED`，保留人工处置流程。 |
| 飞书事件无法准确表达生效日期 | 高 | 提前停用或删除 | 使用生效时间；缺失时仅停用并审批后删除。 |
| 事件重复、乱序 | 高 | 重复创建或误操作 | event_id、账号状态机、任务幂等键和生效时间校验。 |
| 历史账号缺映射 | 高 | 误匹配/漏处理 | 当前不自动处理、不自动删除；上线后以独立历史盘点能力完成确认。 |
| 未结算行程/账单 | 高 | 删除失败或影响结算 | 先停用，删除前查询校验，失败入人工队列。 |
| 凭证泄露/环境混用 | 高 | 外部系统安全风险 | 加密、隔离、允许列表、审计、最小权限、轮换。 |
| 模板和运行时不一致 | 中 | 配置后无法执行 | 模板仅作 UI 起点，运行时使用独立资源与流水线。 |
| 高并发离职 | 中 | 限流、积压 | 限流、队列、退避、告警、压测。 |
| 删除规则误配置 | 高 | 批量误删 | 强校验、审批、灰度、二次确认、默认停用。 |
