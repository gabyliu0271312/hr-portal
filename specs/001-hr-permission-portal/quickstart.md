# Quickstart — 业务用户/IT 小白部署手册

**Feature**: 001-hr-permission-portal · **目标读者**：你（业务用户/HR/IT 小白），不需要懂代码，只需要会复制粘贴和确认输出。

## 0. 你需要准备什么

| 项目 | 说明 | 难度 |
|------|------|------|
| 一台服务器 | Ubuntu 22.04 / Debian 12 / Windows Server 都行；4 核 8G 起步 | 找运维要 |
| 服务器登录方式 | SSH（Linux）或远程桌面（Windows）；账号 + 密码 | 找运维要 |
| 北森 OpenAPI 凭证 | AppKey + AppSecret + 接口地址 | 找北森的对接人要 |
| 飞书自建应用（可选） | 本期先不用，但**最好提前申请下来**避免下一期卡壳 | 找 IT |

## 1. 一行装好运行环境

登录服务器后执行：

```bash
# Linux
curl -fsSL https://get.docker.com | sh
sudo systemctl enable --now docker
```

**怎么验证装好了？** 运行 `docker --version`，能看到版本号就行。

---

## 2. 拉代码 + 启动

```bash
# 找一个你能记住的目录
cd /opt

# 拉代码（实际地址等开发完成后给你）
git clone <repo-url> hr-portal
cd hr-portal

# 配置初始化
cp .env.example .env
# 编辑 .env，至少填这几项（用记事本/vim/nano 都行）：
#   DB_PASSWORD=你设一个强密码
#   JWT_SECRET=随便敲一长串字符
#   ADMIN_INIT_PASSWORD=管理员首次登录密码

# 一行启动所有服务
docker compose up -d
```

**怎么验证启动了？** `docker compose ps` 应该看到三个服务都是 `Up` 状态：

```
NAME                STATUS
hr-portal-db        Up
hr-portal-backend   Up
hr-portal-frontend  Up
```

**健康检查通过前端反向代理执行：**

```bash
curl http://localhost:${FRONTEND_PORT:-8080}/api/v1/health
```

应返回 `"status":"ok"` 且 `db.ok=true`。默认 Compose 不将 HR Portal 后端端口发布到宿主机，因此不会与服务器既有的 `8000` 服务冲突。

**仅开发调试时**，如需从宿主机直接访问 FastAPI 的 `http://localhost:8000/docs`，使用开发端口覆盖：

```bash
docker compose -f docker-compose.yml -f docker-compose.dev.yml up -d --build
```

## 3. 第一次登录

浏览器打开 `http://你服务器的IP/`，应该看到登录页。

- **账号**：admin
- **密码**：你在 .env 里设的 `ADMIN_INIT_PASSWORD`

登录后**第一件事改密码**：右上角头像 → 修改密码。

## 4. 跑通"五件套"——最小可用配置

按这个顺序点一遍，5 张报表就接进来了：

### 4.1 接入北森（系统设置 → 接口配置）

为每张报表新建一个接口：

| 表名 | 刷新模式 | 日期格式 | 业务主键 | 建议定时 |
|------|----------|----------|----------|----------|
| 员工实时花名册 | snapshot | 不带日期 | 员工 ID | 每天 02:00 |
| 员工月度花名册 | upsert | 年月 | 员工 ID + 年月 | 每月 5 号 03:00 |
| 员工月度工资表 | upsert | 年月 | 员工 ID + 年月 | 每月 5 号 03:30 |
| 员工月度成本分摊表 | upsert | 年月 | 员工 ID + 成本中心 + 年月 | 每月 5 号 04:00 |
| 成本中心月度维护表 | upsert | 年月 | 成本中心编码 + 年月 | 每月 5 号 04:30 |

### 4.2 第一次手动拉取一遍

每个接口配置好后，点"立即拉取"，到拉取日志看到 ✅ success 才算成功。**不要五个一起点**——按上表顺序一个个来，避免某张表依赖前一张表的数据没拉到。

### 4.3 树自动生成

5 张表都拉成功后：

- 进入"管理单元 → 组织架构维度" → 应该看到 创梦天地 → 公司级 → 一~五级部门 的树
- 进入"管理单元 → 成本中心维度" → 应该看到一~四级成本中心树

如果树是空的，先在右上角"重新生成树"刷新一下。

### 4.4 建第一个角色 + 第一个用户

- "系统设置 → 角色配置" → 新建角色 "HR-财务对账"，勾选要给的菜单
- "系统设置 → 用户管理" → 新建用户，绑定上面那个角色 + 至少给一个数据范围标签

### 4.5 建第一个数据集 + 第一份报表

- "报表设置 → 表间关联" → 新建数据集 "月度人力成本"，把 4 张月度表加进去，按"员工 ID + 年月"建关联
- "报表管理 → 新建报表" → 第一步选刚建的数据集 → 拖维度（成本中心、用工类型）→ 拖度量（工资合计）→ 预览 → 保存

走到这里整套系统就跑通了。

## 5. 怎么备份数据

```bash
# 备份（每天凌晨自动跑也行，加到 cron）
docker exec hr-portal-db pg_dump -U hr_portal hr_portal > backup_$(date +%Y%m%d).sql

# 恢复
docker exec -i hr-portal-db psql -U hr_portal hr_portal < backup_20260601.sql
```

**重要**：备份文件**别放在服务器同一块盘**！至少同步到一个云盘或另一台机器。

## 6. 出问题怎么办

| 现象 | 怎么排查 |
|------|----------|
| 网站打不开 | `docker compose ps` 看哪个服务挂了，`docker compose logs <服务名>` 看日志 |
| 拉北森报错 | 接口配置 → 拉取日志 → 看错误详情；80% 是凭证过期或字段映射没对上 |
| 用户登录后看不到该看的数据 | 检查该用户是否绑定了"成本中心维度"和"组织架构维度"两类标签——少任一类都会被卡住（FR-AUTH-002） |
| 报表打开报"关联失效" | 数据集详情页 → 检查关联是否还能打通；可能是源表字段被改名了 |
| 服务器越来越慢 | 大概率是日志 / 备份占满磁盘，先 `df -h` 看下 |
