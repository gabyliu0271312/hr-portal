接入系统在 UCP 中起什么作用

把外部 HR 系统（北森 / 飞书 / 滴滴 / OA）变成 UCP 可调用、可编排、可监控的标准动作

核心价值

统一协议 + 解耦凭证 + 事件驱动 + 可视化编排 + 全链路可观测

4 个核心实体

按依赖顺序排列，左侧是右侧的输入

典型配置流程（一次接入北森）

  1. 凭证：保存北森 AppKey / AppSecret（加密落库，可复用）
  2. 连接器：选「北森-HR 系统」，绑定凭证，配 base_url / 超时 / 重试策略
  3. 适配器：选 BEISEN_EMPLOYEE_ADAPTER，配置字段映射 + 脱敏规则
  4. 流水线：拉北森员工 → 适配器清洗 → 推到飞书（多步编排）
  5. 触发器 + 监控：监听飞书/北森事件自动执行流水线，监控中心看执行日志与告警
每张「接入系统」卡片 = 1 个 Connector（按 system_code 聚合），展示凭证 / 连接器 / 流水线 / 24h 事件数量

你每天会在接入系统页看到

  • 已配置系统的卡片网格（北森/飞书/滴滴…）
  • 每张卡片：连接状态 + 配置数量
  • 点击卡片 → 详情抽屉，配置 / 测试 / 启用停用

什么时候需要回到这个页

  • 新增一个外部系统（先建凭证再建连接器）
  • 外部系统密钥轮换（编辑凭证）
  • 新接入 HR 系统时，按向导跑一遍 5 步流程
