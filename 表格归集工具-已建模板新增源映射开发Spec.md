## 0. 与既有文档的关系与现状

### 0.1 文档关系

本文件是 `006 — 表格处理工具（表格合并 / 拆分）` 的**增量开发 Spec**，用于落实其“模板管理页维护各模板映射配置与版本”的目标。它不替代主 Spec 中的归集模型、AI 安全红线、合并引擎和权限原则。

本需求的定义是：**在模板已创建后，管理员可为同一标准字段模板新增一种源 Excel 表结构映射，并可继续编辑或删除该映射。**它不是新增业务数据库表，也不是新增归集输出字段。

开发阅读顺序：`AGENTS.md` → `specs/006-excel-table-tools/spec.md` → 本文件 → `backend/app/table_tools/{router,models,engine}.py` → `frontend/src/views/tools/TableMerge.vue` → `frontend/src/api/tableTools.ts`。

### 0.2 当前实现梳理（以代码为准）

| 范围 | 已实现 | 当前缺口 |
|---|---|---|
| 数据模型 | `MergeTemplate` 与 `MergeSourceMapping` 已是一对多；源映射已含名称、表头特征、Sheet、表头区间、主键/字段映射和派生字段。 | MVP 无需新增表或字段。 |
| 模板读取/整体更新 | `GET /templates/{tid}` 返回全部映射；`PUT /templates/{tid}` 可保存完整 `mappings`。 | 当前更新会清空并重建全部映射，不宜用于长期单条维护。 |
| 前端映射编辑 | 可编辑 Sheet、表头行、主键映射、字段映射和派生字段。 | 没有“新增映射表”入口；未编辑映射名称与 `match_signature`；没有样表表头预览。 |
| AI 建模板 | `/table-tools/ai-draft` 能生成完整模板草稿。 | 不应直接用于追加映射，避免改写既有标准字段/主键。 |
| 运行识别 | 引擎按表头特征、Sheet、表头区间选取最佳映射。 | 未命中 Sheet 当前会静默跳过，需补异常可见性。 |
| 版本 | 修改模板时 `version += 1`。 | 没有版本快照和回滚，本期不建设。 |

## 1. 背景与目标

- **背景**：同一归集场景会不断接入新地区、供应商或系统导出的 Excel 格式；它们应映射至已有模板的标准字段，而非重复创建完整模板。
- **目标**：
  1. 管理员可在已有模板内新增、编辑、删除单条源映射。
  2. 支持上传代表性样表解析表头并生成草稿，也支持完全手动配置。
  3. 新映射保存后参与自动识别；未匹配/歧义匹配必须在异常列表中可见。
  4. 不影响既有模板整体编辑、合并、下载和旧数据读取。
- **非目标 / 不做范围**：不修改标准字段、归集主键和聚合口径；不实现模板快照/回滚；不发送 Excel 明细给 AI；不涉及 UCP、第三方系统、动态 SQL 或表格拆分。

## 2. 用户场景

### 2.1 模板管理员：新增源映射（样表主路径）

- **入口**：`/tools/table-merge` → 编辑模板 → 数据源映射区 → `新增映射表`。
- **操作**：输入映射名称，上传样表，选择 Sheet 与表头起止行；确认解析表头和表头特征；设置主键映射、字段映射、派生字段、跳过词。
- **系统反馈**：解析中显示加载态；无可用 Sheet、表头为空、文件不合法时提示原因；低置信 AI 建议标黄但允许人工修改。
- **成功**：保存单条映射后出现在列表，模板版本号加一；后续同格式文件可命中。
- **失败/无权限**：名称重复、特征少于三项、目标字段非法时阻止提交；非创建者且非超管不可见新增/编辑/删除入口。

### 2.2 模板管理员：手动新增/编辑/删除

- **入口**：新增映射表选择“手动配置”；已有映射卡片点击“编辑”或“删除”。
- **操作**：手动输入至少三个表头特征，维护映射规则；删除前二次确认。
- **结果**：编辑/删除只影响目标映射，不删除或重建其他映射。
- **异常**：`tid`/`mid` 不存在返回 404；无权限 403；保存失败保留前端已填内容。

### 2.3 普通使用者：运行合并

- **入口/操作**：选择模板后上传文件并合并。
- **反馈**：展示命中的文件、Sheet、映射名和分数；未命中或歧义 Sheet 必须出现异常项，禁止静默丢弃。
- **权限**：只有运行权限的用户不具备模板维护入口。

## 4. 技术设计

### 4.1 数据库 / 数据模型

- **表与 Migration**：复用 `merge_templates`、`merge_source_mappings`，本期不新增表、列或 Alembic migration；因此无 downgrade。
- **兼容**：旧映射读取、运行保持兼容；只有用户新建或编辑该映射时才执行新的写入校验。
- **单条标识**：使用现有 `merge_source_mappings.id` 作为编辑/删除资源标识；不得通过模板整体 `PUT` 的清空重建逻辑替代单条操作。
- **版本**：新增、更新、删除映射均在同一事务内执行父模板 `version += 1`。该版本号不等于历史快照。

| 字段 | 写入校验 |
|---|---|
| `name` | 必填，1–128 字符，同模板唯一。 |
| `match_signature` | 去空白、去重后至少 3 个有效表头。 |
| `sheet_kw` | 可空，非空最大 128 字符。 |
| `header_start/header_end` | `1 <= start <= end <= 10`。 |
| `key_map` | 至少一项；目标只能属于父模板 `merge_keys`。 |
| `column_map` | 源列非空；目标只能属于父模板 `std_fields`。 |
| `derived_fields[].target` | 只能属于父模板 `std_fields`。 |
| `skip_tokens` | 默认 `['合计','小计','总计']`，去空值、去重。 |

### 4.2 后端接口

既有 `GET /api/v1/table-tools/templates/{tid}` 与 `PUT /api/v1/table-tools/templates/{tid}` 保持兼容；整体更新也应复用本期的映射校验器。

| URL / Method | Request | Response | 权限 | 状态码/错误 |
|---|---|---|---|---|
| `POST /api/v1/table-tools/templates/{tid}/mapping-draft` | `multipart/form-data`：`file` 必填，`sheet_name`、`header_start`、`header_end`、`business_context` 可选 | `mapping` 草稿、`available_sheets`、`effective_headers`、`low_confidence`、`warnings` | `table_tools:U` + owner/超管 | 400 文件/参数错误，403，404，422 表头不可解析，503 AI 不可用。 |
| `POST /api/v1/table-tools/templates/{tid}/mappings` | `SourceMappingIn` | 201，完整映射（含 `id`） | 同上 | 400/422 校验，403，404，409 名称重复。 |
| `PUT /api/v1/table-tools/templates/{tid}/mappings/{mid}` | `SourceMappingIn` | 200，完整映射 | 同上 | 403，404，409。 |
| `DELETE /api/v1/table-tools/templates/{tid}/mappings/{mid}` | 无 | 204 | 同上 | 403，404。 |

`mapping-draft` 响应中的草稿必须满足现有 `SourceMappingIn` 字段结构；`key_map`、`column_map` 和派生字段目标只能引用父模板已存在的主键/标准字段。草稿不自动落库。

合并接口 URL 不变。非空 Sheet 未匹配时，在 `anomalies` 返回：

```json
{"type":"未匹配源映射","key":"文件名.xlsx / Sheet1","detail":"未找到符合表头特征和 Sheet 规则的源映射","file":"文件名.xlsx"}
```

同分且无法区分的多个映射返回“映射命中歧义”异常，不随机选择。

### 4.3 业务逻辑

1. 后端读取父模板，先执行 `U` 和创建者/超管门禁。
2. 样表解析复用 `app.table_tools.engine` 的多行表头解析；只读取 Sheet、表头和结构统计，不将明细入 AI 上下文。
3. AI 建议模式复用现有受控 capability / policy guard / schema / audit 链路，只生成一条 `SourceMapping` 草稿；AI 不可用时允许手动模式继续。
4. 实现统一 `validate_source_mapping(template, payload)`，由单条 CRUD 和既有整体 `PUT` 共用；在一个事务内保存映射并递增模板版本，失败整体回滚。
5. 保留现有最佳匹配逻辑；补齐未匹配与歧义异常，正常命中结果不变。
6. 禁止重复建设：不新写 Excel 解析器、不私接 AI Provider、不新写公式求值器；分别复用 `table_tools.engine`、既有 AI 底座、`app.ai_formula`。

### 4.4 前端与 UI/交互

- **页面**：`hr-portal/frontend/src/views/tools/TableMerge.vue`，路径 `/tools/table-merge`。
- **入口**：编辑模板的“数据源映射”标题右侧增加 `新增映射表`；仅具备 `table_tools:U` 且通过 owner/超管判断时显示。
- **编辑容器**：使用右侧抽屉（推荐）或对话框，标题为“新增映射表/编辑映射表”。

| 区域 | 字段/交互 |
|---|---|
| 基本信息 | 映射名称；实时提示同模板重复名。 |
| 配置方式 | “从样表生成”“手动配置”；默认前者。 |
| 样表解析 | 上传、Sheet 下拉、表头起止行、重新解析、有效表头标签预览。 |
| 自动识别特征 | `match_signature` 标签多选/可输入；至少三项。 |
| 主键/字段映射 | 源列→目标字段；目标下拉只来自当前模板 `merge_keys` / `std_fields`。 |
| 派生字段/其他 | 目标字段、公式、精度、Sheet 关键词、跳过词、拆分校验。 |
| 操作 | 取消、保存映射；成功刷新列表和版本，失败保留表单。 |

- **列表**：卡片展示名称、Sheet、表头范围、特征数、字段映射数；每项有编辑/删除，删除必须二次确认并说明不影响其他映射。
- **状态**：解析中禁重复提交；AI 建议中提示“仅分析表头”；低置信标黄；空态提示可新增映射；403 提示仅创建者/超管可维护。

### 4.5 权限、安全与外部系统

| 操作 | 权限与边界 |
|---|---|
| 查看模板/映射 | `table_tools:V`。 |
| 新增、编辑、删除、生成草稿 | `table_tools:U` 且模板创建者或超级管理员。 |
| 合并/下载 | 沿用现有 `V/E`，本期不改变。 |

- 样表只在服务端临时解析；AI 只接收有效表头、Sheet、结构元数据及父模板字段，严禁姓名、证件号、金额等数据行、文件原文出网。
- AI 结果必须经过 capability schema、policy guard、输出审查和审计；用户确认前不得写库。
- 派生公式继续受公共公式执行器限制；禁止动态 SQL、脚本、URL 或自定义代码执行。
- 不涉及 UCP、连接器、外部凭证或第三方系统。

## 7. 验收标准

- **用户**：管理员无需新建模板即可新增源映射；新增格式可自动归集；AI 不可用仍可手工完成。
- **开发**：单条 CRUD 使用稳定 `mid`，统一校验，接口与状态码符合第 4.2 节，未命中/歧义可追踪。
- **测试**：第 6 节场景有自动化或可复现手工证据；后端测试和前端构建通过。
- **UI/交互**：入口、抽屉、加载、低置信、空态、失败、无权限、删除确认全部符合第 4.4 节；失败不丢表单。
- **上线**：无敏感明细出网路径；无 migration 部署风险；已明确本期不支持版本回滚。

## 8. 风险与兼容性

| 风险 | 等级 | 影响 | 应对 |
|---|---|---|---|
| 表头特征不稳定 | 高 | 误判或无法命中。 | 至少三项特征；未命中/歧义异常可见。 |
| 完整模板草稿覆盖旧模板 | 高 | 标准字段或旧映射被改写。 | 只生成单条映射草稿，受父模板约束。 |
| 全量 PUT 清空重建 | 高 | ID/创建时间丢失、并发覆盖。 | 新增单条 CRUD，前端不以整体 PUT 保存单条。 |
| AI 接收明细 | 高 | 敏感信息泄露。 | 仅传表头与结构元数据，强制 guard/audit。 |
| 旧映射不符合新校验 | 中 | 首次编辑无法保存。 | 读取兼容，仅写入校验，前端引导修复。 |
| 无版本快照 | 中 | 误改不可一键回滚。 | 本期仅递增版本号；后续另立快照需求。 |

### 8.1 假设与待确认事项

1. “新增映射表格”指新增一种源 Excel 结构映射。
2. 标准字段、主键和聚合口径不在本需求的新增流程中修改。
3. 以现有实际权限 `table_tools:U` 作为维护权限；若后续改为 `manage_template`，须同步接口和前端判断。
4. 本期接受版本号递增但不支持历史回滚。
